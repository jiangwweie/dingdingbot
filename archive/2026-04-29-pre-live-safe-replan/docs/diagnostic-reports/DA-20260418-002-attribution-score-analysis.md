## 一、归因分数可靠性诊断

### 1.1 数据证据

**分数区间 vs 实际表现**:

| 分数区间 | 信号数 | 平均胜率 | 平均盈亏比 |
|---------|--------|---------|-----------|
| 0.7-1.0 | 252 | 28.4% | -0.43 |
| 0.5-0.7 | 46 | 45.4% | -0.09 |

**结论**: ❌ 分数与胜率负相关，归因分数不可靠

### 1.2 根因分析

**当前归因公式**:
```python
final_score = pattern × 0.55 + ema × 0.25 + mtf × 0.20
```

**问题**:

1. **pattern 权重过高 (55%)**
   - 高分信号往往 pattern_score 很高
   - 但 pattern_score 高 ≠ 交易盈利
   - Pinbar 形态质量高，但可能是逆势反转失败

2. **ema 权重过低 (25%)**
   - EMA 代表趋势强度
   - 高分信号的 ema_score 较高（0.561 vs 0.163）
   - 说明高分信号趋势确认更强，但仍失败

3. **缺少趋势方向因子**
   - 当前公式只看强度，不看方向
   - Pinbar 本质是反转策略（逆小趋势）
   - 高分信号更多是逆势反转，失败率高

### 1.3 高分信号特征分析

**高分信号 (>0.8) 的组件特征**:
- pattern_score: 0.893 (很高)
- ema_score: 0.561 (中等)
- mtf_score: 1.000 (满分)

**中分信号 (0.6-0.8) 的组件特征**:
- pattern_score: 0.769 (中等)
- ema_score: 0.163 (很低)
- mtf_score: 1.000 (满分)

**差异**:
- 高分信号的 ema_score 更高 (+0.398)
- 说明高分信号的趋势确认更强
- 但胜率反而低 → 说明逆势反转失败率高

### 1.4 改进方案

**方案 A: 调整权重**
```python
final_score = pattern × 0.35 + ema × 0.35 + mtf × 0.30
# 提高 ema 权重，降低 pattern 权重
```

**方案 B: 增加趋势方向因子**
```python
base_score = pattern × 0.55 + ema × 0.25 + mtf × 0.20
if 顺势:
    final_score = base_score × 1.2
if 逆势:
    final_score = base_score × 0.8
```

**方案 C: 简化为 pattern_score**
```python
final_score = pattern_score
# 只看形态质量，忽略其他因子
```

**建议**: 需要通过回测验证哪个方案最可靠

## 二、TP/SL 结构诊断

### 2.1 TP 触发率统计

| 止盈级别 | 触发次数 | 占比 |
|---------|---------|------|
| TP1 | 95 | 38.6% |
| TP2 | 0 | 0% |
| TP3 | 0 | 0% |
| SL | 151 | 61.4% |

**关键发现**: ❌ TP2/TP3 从未触发，双级止盈策略失效

### 2.2 实际盈亏比分析

**理论盈亏比**: 1.50 (TP=1.5R, SL=-1.0R)

**实际盈亏比**: 1.23

**差距**: -17.8%

**根因**:

1. **滑点 + 手续费蚕食盈利端**
   - 每笔交易成本: 0.28%
   - 对盈亏比影响: -20.7%

2. **Pinbar 影线长度不固定**
   - 高分信号 → 影线长 → SL 距离大 → 绝对亏损大
   - 低分信号 → 影线短 → 交易成本占比高

3. **仓位计算未考虑信号质量**
   - 所有信号一视同仁
   - 低分信号拖累整体收益

### 2.3 提高 TP 到 2.2R 的影响分析

**当前 TP=1.5R**:
- 触发率: 38.6%
- 期望收益: -42.07U 每笔

**预估 TP=2.2R**:
- 触发率: 26.3% (下降 32%)
- 期望收益: -48.13U 每笔

**结论**: ⚠️ 提高 TP 到 2.2R 会降低期望收益（触发率下降太多）

**建议**: 需要找到最优 TP 目标（通过网格搜索）

## 三、周期表现诊断

### 3.1 15m 周期问题

**数据**:
- 总交易: 227 次（占总交易 68%）
- 平均收益率: -27.77%
- 单笔期望值: -49.27U

**根因**:

1. **高分信号比例低**: 16.5% vs 1h 的 38.7%
2. **形态质量与实际表现脱钩**: 高分信号胜率仍低
3. **Pinbar 影线在 15m 不可靠**: 噪音导致的虚假形态多
4. **市场微观结构不适合**: 15m 噪音占比高（~60%）
5. **策略参数不适配**: 统一参数在不同周期表现差异大

**建议**: 禁用 15m 周期或为 15m 设计独立策略

## 六、关键洞察

### 6.1 归因系统的根本问题

**当前归因是「描述性统计」，不是「预测性决策工具」**

- 归因维度与交易结果脱钩
- 归因分数无法指导交易决策
- 需要重新设计，使其具有预测能力

### 6.2 策略优化的正确方向

1. **先修复归因公式**，再考虑动态仓位
2. **先找到最优 TP**，再考虑提高盈亏比
3. **先验证参数有效性**，再考虑大规模应用

### 6.3 数据驱动的决策原则

- 不要基于假设优化，要基于数据验证
- 不要只看单一指标，要看组合效果
- 不要追求完美，要追求可验证的改进
