# 盯盘狗 🐶 v1.0.0 正式发布

**发布日期**: 2026-03-29
**版本**: v1.0.0
**阶段**: 第一阶段完成 - 架构筑基

---

## 🎉 发布概述

盯盘狗 v1.0.0 是一个**完全动态化、高并发、强状态一致性**的加密货币量化交易信号监控与回测沙箱系统。

**核心原则**: Zero Execution Policy（零执行政策）- 系统仅为观测与通知工具，严禁集成任何交易下单接口。

---

## ✨ 核心功能

### 1. 动态策略系统

- **积木化策略工作台** - 零代码前端界面，自由拼装触发器与过滤器
- **多策略并行** - 支持多个策略同时运行，独立作用域配置
- **动态热重载** - 配置文件修改后自动生效，无需重启进程

**支持的触发器**:
| 触发器 | 说明 | 参数 |
|--------|------|------|
| Pinbar | 针 bar 形态检测 | min_wick_ratio, max_body_ratio, body_position_tolerance |
| Engulfing | 吞没形态检测 | min_body_ratio, require_full_engulf |
| Doji | 十字星形态 | max_body_ratio, min_total_range |
| Hammer | 锤子线形态 | min_lower_wick_ratio, max_upper_wick_ratio |

**支持的过滤器**:
| 过滤器 | 说明 | 参数 |
|--------|------|------|
| EMA | EMA 趋势校验 | period, trend_direction |
| MTF | 多周期趋势验证 | require_confirmation |
| ATR | ATR 波动率过滤 | period, min_atr_ratio |
| Volume Surge | 成交量激增检测 | multiplier, lookback_periods |
| Volatility Filter | 波动率过滤 | min_atr_ratio, max_atr_ratio, atr_period |
| Time Filter | 时间窗口过滤 | session, exclude_weekend |
| Price Action | 价格行为过滤 | min_body_size, max_body_size |

### 2. 多级别止盈（S6-3）

- **动态止盈配置** - 支持多个止盈级别（TP1, TP2, TP3...）
- **独立仓位比例** - 每个止盈级别可配置独立的仓位比例
- **实时状态跟踪** - 每个止盈级别独立跟踪状态（PENDING/WON/CANCELLED）

**配置示例**:
```yaml
take_profit_levels:
  - tp_id: TP1
    position_ratio: 0.5    # 50% 仓位
    risk_reward: 1.5       # 1:1.5 盈亏比
  - tp_id: TP2
    position_ratio: 0.3    # 30% 仓位
    risk_reward: 3.0       # 1:3 盈亏比
  - tp_id: TP3
    position_ratio: 0.2    # 20% 仓位
    risk_reward: 5.0       # 1:5 盈亏比
```

### 3. 信号覆盖系统（S6-2）

- **信号评分优化** - Pinbar 形态质量评分（0-1）
- **信号覆盖机制** - 更优信号自动替代旧信号，避免重复通知
- **反方向信号检测** - 检测并提示反方向信号出现

**评分标准**:
| 分数范围 | 质量等级 | 说明 |
|----------|----------|------|
| 0.8 - 1.0 | 优秀 | 形态完美，强烈关注 |
| 0.6 - 0.8 | 良好 | 形态标准，可关注 |
| 0.5 - 0.6 | 一般 | 形态勉强，谨慎观察 |
| < 0.5 | 较差 | 形态不合格，建议忽略 |

### 4. 回测沙箱系统

- **即时回测** - 支持自定义时间范围的回测
- **策略预览** - 实盘前 Dry Run 即时验证
- **性能统计** - 胜率、盈亏比、过滤统计等详细报告
- **信号持久化** - 回测信号自动保存到数据库

**回测指标**:
- 总信号数 / 总过滤数
- 各过滤器过滤统计
- 胜率 / 亏损率
- 总盈亏 / 平均盈亏
- 执行时间 / K 线分析数

### 5. 实时通知系统

- **多渠道通知** - 飞书、企业微信、Telegram
- **动态标签** - 信号携带过滤器状态标签（MTF、EMA 等）
- **消息增强** - 包含评分、覆盖率、盈亏比等详细信息

**通知示例**:
```
【交易信号提醒】

币种：BTC/USDT:USDT
周期：1h
方向：🟢 看多 (LONG)
入场价：68500
止损位：68000
建议仓位：0.52 BTC
当前杠杆：5x

指标标签:
  MTF: Confirmed
  EMA: Bullish
  ATR: Normal

风控信息：Risk 1.00% = 100.00 USDT
形态评分：0.78

---
⚠️ 本系统仅为观测与通知工具，不构成投资建议
```

### 6. 信号全生命周期跟踪（S5-2）

- **状态枚举** - GENERATED → PENDING → ACTIVE → FILLED/WON/LOST
- **实时更新** - 价格触及止盈/止损时自动更新状态
- **历史追溯** - 完整的信号状态变更历史

### 7. 配置快照版本化（S5-3）

- **快照管理** - 支持配置版本保存、回滚
- **版本对比** - 可视化对比不同版本差异
- **一键恢复** - 快速回滚到历史版本

## 📦 技术架构

### 系统分层 (Clean Architecture)

```
┌─────────────────────────────────────────────────┐
│             interfaces/ (REST API)               │
│         FastAPI 端点：/api/signals, etc.         │
├─────────────────────────────────────────────────┤
│          application/ (应用服务层)                │
│   signal_pipeline, backtester, config_manager   │
├─────────────────────────────────────────────────┤
│           domain/ (领域核心层，纯业务)             │
│  models, strategy_engine, filter_factory, risk  │
├─────────────────────────────────────────────────┤
│        infrastructure/ (基础设施层，I/O)          │
│  exchange_gateway, notifier, logger, repository │
└─────────────────────────────────────────────────┘
```

### 技术栈

| 领域 | 技术 |
|------|------|
| **后端语言** | Python 3.11+ |
| **Web 框架** | FastAPI + asyncio + Uvicorn |
| **交易所** | CCXT (async_support) + CCXT.Pro (WebSocket) |
| **数据验证** | Pydantic v2 |
| **金融精度** | `decimal.Decimal`（严禁 `float`） |
| **数据库** | SQLite (aiosqlite) |
| **测试** | pytest + pytest-asyncio + pytest-cov |
| **前端** | React 18 + TypeScript 5 + TailwindCSS 3 |
| **图表** | Lightweight Charts |
| **状态管理** | SWR + React Hooks |

### 核心约束

| 约束 | 说明 |
|------|------|
| **领域层纯净性** | `domain/` 严禁导入 ccxt、aiohttp、fastapi、yaml |
| **Decimal everywhere** | 所有金额、比率计算必须使用 `decimal.Decimal` |
| **API 密钥安全** | 必须只读权限，启动时校验，敏感信息脱敏 |
| **类型安全** | 禁用 `Dict[str, Any]`，使用 Pydantic 具名类 |

## 🚀 快速开始

### 1. 安装依赖

```bash
# 创建虚拟环境
python3 -m venv venv
source venv/bin/activate

# 安装依赖
pip install -r requirements.txt
```

### 2. 配置系统

**核心配置** (`config/core.yaml`) - 只读，系统默认值
**用户配置** (`config/user.yaml`) - 可编辑，覆盖默认值

```yaml
# config/user.yaml
exchange:
  name: binance
  api_key: YOUR_READ_ONLY_API_KEY    # ⚠️ 必须只读权限
  api_secret: YOUR_API_SECRET
  testnet: true                       # 测试网

user_symbols: []                      # 空时使用 core_symbols
timeframes:
  - 15m
  - 1h

active_strategies:
  - id: strategy-001
    name: 01pinbar-ema60
    is_global: true
    logic_tree:
      gate: AND
      children:
        - type: trigger
          config:
            type: pinbar
            params:
              min_wick_ratio: 0.5
              max_body_ratio: 0.35
        - type: filter
          config:
            type: ema
            params:
              period: 60
        - type: filter
          config:
            type: mtf
            params:
              require_confirmation: true

risk:
  max_loss_percent: '0.02'            # 单笔最大亏损 2%
  max_leverage: 125
  max_total_exposure: '0.8'           # 最大仓位 80%

notification:
  channels:
    - type: feishu
      webhook_url: https://open.feishu.cn/open-apis/bot/v2/hook/xxx
```

### 3. 运行系统

```bash
# 运行主程序（实时监控 + REST API）
python src/main.py

# 访问 REST API
# GET  http://localhost:8000/api/signals     - 查询历史信号
# POST http://localhost:8000/api/backtest    - 运行回测
# GET  http://localhost:8000/api/strategies  - 获取策略模板
# GET  http://localhost:8000/api/config      - 获取配置

# 访问前端
# 开发模式：cd gemimi-web-front && npm run dev
# 访问：http://localhost:5173
```

### 4. 运行回测

```bash
python tests/backtest.py
```

### 5. 运行测试

```bash
# 安装测试依赖
pip install pytest pytest-asyncio pytest-cov

# 运行单元测试
pytest tests/unit/ -v

# 运行覆盖率测试
pytest tests/unit/ --cov=src --cov-report=html
```

## 📝 API 接口文档

### 信号相关

| 端点 | 方法 | 说明 |
|------|------|------|
| `/api/signals` | GET | 查询信号列表 |
| `/api/signals/{id}` | GET | 查询单个信号详情 |
| `/api/signals/{id}/context` | GET | 查询信号上下文 K  线 |
| `/api/signals/{id}/status` | GET | 查询信号状态 |
| `/api/signals/status` | GET | 查询所有信号状态 |
| `/api/signals` | DELETE | 批量删除信号 |

### 回测相关

| 端点 | 方法 | 说明 |
|------|------|------|
| `/api/backtest` | POST | 运行回测 |

| 端点 | 方法 | 说明 |
|------|------|------|

| `/api/strategies/templates` | GET | 获取策略模板 |
| `/api/strategies/{id}/apply` | POST | 应用策略到实盘 |
| `/api/strategies/preview` | POST | 预览策略（Dry Run） |
| `/api/strategies/meta` | GET | 获取策略元数据 |

### 配置相关

| 端点 | 方法 | 说明 |
|------|------|------|
| `/api/config` | GET | 获取系统配置 |
| `/api/config` | PUT | 更新系统配置 |

## 🔧 错误码系统

| 错误码 | 级别 | 说明 |
|--------|------|------|
| `F-001` | FATAL | API Key 有交易权限 |
| `F-002` | FATAL | API Key 有提现权限 |
| `F-003` | FATAL | 必填配置缺失 |
| `F-004` | FATAL | 交易所初始化失败 |
| `C-001` | CRITICAL | WebSocket 重连超限 |
| `C-002` | CRITICAL | 资产轮询连续失败 |
| `W-001` | WARNING | K  线数据质量异常 |
| `W-002` | WARNING | 数据延迟超限 |
