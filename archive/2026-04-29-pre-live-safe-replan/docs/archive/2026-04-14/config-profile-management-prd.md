## 一、产品概述

### 1.1 产品愿景

**配置 Profile（配置档案）** = 一套完整的策略参数 + 风控配置 + 交易所配置的组合

允许用户同时维护多套配置档案，根据市场环境/交易风格快速切换，无需手动备份还原。

### 1.2 核心价值

| 用户痛点 | 解决方案 | 价值体现 |
|----------|----------|----------|
| 想同时维护激进/保守两套配置 | 创建多个 Profile，一键切换 | 快速适应市场变化 |
| 配置修改后无法快速回退 | Profile 切换 + 快照版本化 | 降低试错成本 |
| 配置备份麻烦，分享困难 | YAML 导出/导入 | 易于迁移和团队协作 |
| 不确定配置变更影响 | 切换前预览差异 | 透明决策，减少误操作 |

### 1.3 目标用户

| 用户类型 | 使用场景 | 核心需求 |
|----------|----------|----------|
| 多策略交易员 | 同时运行趋势 + 均值回归策略 | 快速切换策略配置 |
| 风险敏感型用户 | 根据行情调整风险敞口 | 激进/保守快速切换 |
| 多时段交易者 | 亚盘/欧盘/美盘不同配置 | 定时/手动切换 |
| 团队管理者 | 分享配置给团队成员 | 导出/导入配置 |

#### F1: Profile 列表查看 (P0)

**用户故事**:
> 作为用户，我想查看所有配置档案列表及其状态，了解当前激活的是哪个。

**功能描述**:
- 展示所有 Profile 列表（名称、描述、创建时间、更新时间）
- 标识当前激活的 Profile（高亮/徽章）
- 显示每个 Profile 的配置项数量
- 支持按名称搜索/过滤

**界面元素**:
| 元素 | 说明 |
|------|------|
| Profile 名称 | 可点击，进入详情 |
| 激活状态徽章 | 🟢 激活中 / ⚪ 未激活 |
| 描述 | 简短说明（20 字内） |
| 创建时间 | ISO 8601 格式 |
| 更新时间 | 最近一次修改时间 |
| 配置项数量 | 该 Profile 下的配置项总数 |

**验收标准**:
- [ ] 列表加载时间 < 500ms
- [ ] 激活 Profile 有明显视觉标识
- [ ] 支持空状态提示（仅 default 时）

#### F2: 创建 Profile (P0)

**用户故事**:
> 作为用户，我想从当前配置创建一个新档案，以便尝试新的参数组合。

**功能描述**:
- 从当前激活的 Profile 复制所有配置项
- 用户输入新 Profile 名称和描述
- 名称验证：唯一性检查（不能与现有重复）
- 创建后自动切换到新 Profile（可选）

**创建流程**:
```
1. 用户点击"新建 Profile"按钮
         ↓
2. 弹出创建对话框
   - 名称输入框（必填，1-32 字符）
   - 描述输入框（可选，0-100 字符）
   - 复选框："创建后立即切换"（默认勾选）
         ↓
3. 前端验证
   - 名称不能为空
   - 名称不能包含特殊字符
   - 名称不能与现有重复（调用 API 检查）
         ↓
4. 调用后端创建 API
         ↓
5. 创建成功
   - 关闭对话框
   - 刷新列表
   - 如勾选则触发切换
```

**名称验证规则**:
| 规则 | 说明 | 错误提示 |
|------|------|----------|
| 长度 | 1-32 字符 | "名称长度为 1-32 个字符" |
| 字符合法性 | 允许字母/数字/中文/下划线/连字符 | "名称只能包含字母、数字、中文、下划线或连字符" |
| 唯一性 | 不能与现有 Profile 重名 | "该名称已被使用" |
| 保留字 | 不能使用 'default' | "'default' 是保留名称" |

**验收标准**:
- [ ] 创建时间 < 1s
- [ ] 名称实时验证（防抖 300ms）
- [ ] 创建成功后有成功提示

#### F3: 切换 Profile (P0)

**用户故事**:
> 作为用户，我想在切换前看到配置差异，确认无误后再切换。

**功能描述**:
- 切换前弹出预览对话框，显示配置差异
- 差异按模块分组展示（策略/风控/交易所）
- 用户确认后执行原子切换
- 切换后触发配置热重载

**预览对话框内容**:
```

│  切换配置预览                                    [×]    │
├─────────────────────────────────────────────────────────┤
│  当前：保守型                                             │
│  目标：激进型                                             │
│                                                         │
│  差异总览：共 15 项配置不同                                │
│                                                         │
│  ─────────────────────────────────────────────────────  │
│  📊 策略配置 (8 项)                                       │
│  ─────────────────────────────────────────────────────  │
│  1. Pinbar.min_wick_ratio                               │
│     0.6 → 0.5                                           │
│  2. Pinbar.max_body_ratio                               │
│     0.3 → 0.35                                          │
│  3. EMA.period                                          │
│     50 → 20                                             │
│  4. EMA.trend_filter_enabled                            │
│     开启 → 关闭                                         │
│  ... (显示前 5 项，其余折叠)                               │
│                                                         │
│  ─────────────────────────────────────────────────────  │
│  ⚠️ 风控配置 (5 项)                                       │
│  ─────────────────────────────────────────────────────  │
│  1. max_loss_percent                                    │
│     1.0% → 2.0%                                         │
│  2. trailing_stop_percent                               │
│     1.5% → 3.0%                                         │
│  3. take_profit_levels                                  │
│     2 级 → 3 级                                          │
│  ...                                                    │
│                                                         │
│  ─────────────────────────────────────────────────────  │
│  🔌 交易所配置 (2 项)                                     │
│  ─────────────────────────────────────────────────────  │
│  1. leverage                                            │
│     5x → 10x                                            │
│  2. default_symbols                                     │
│     [BTC,ETH] → [BTC,ETH,SOL,BNB]                       │
│                                                         │
│  [取消]                        [确认切换]                 │
└─────────────────────────────────────────────────────────┘
```

**切换流程**:
```
1. 用户点击列表中的"切换"按钮
         ↓
2. 前端调用差异对比 API
         ↓
3. 弹出预览对话框（显示差异）
         ↓
4. 用户点击"确认切换"
         ↓
5. 调用切换 API（后端原子操作）
         ↓
6. 切换成功
   - 关闭对话框
   - 刷新列表
   - 显示成功提示
   - 自动刷新页面配置（热重载）
```

**验收标准**:
- [ ] 差异计算时间 < 500ms
- [ ] 差异展示清晰，按模块分组
- [ ] 切换操作 < 1s
- [ ] 切换后配置立即生效（无需重启）
- [ ] 禁止删除当前激活的 Profile

#### F4: 删除 Profile (P1)

**用户故事**:
> 作为用户，我想删除不再需要的配置档案，保持列表整洁。

**功能描述**:
- 删除前二次确认
- 禁止删除当前激活的 Profile
- 禁止删除 default Profile
- 删除后不可恢复（软删除可选）

**删除流程**:
```
1. 用户点击"删除"按钮
         ↓
2. 检查删除条件
   - 是否是 default？ → 禁止，提示
   - 是否是当前激活？ → 禁止，提示
         ↓
3. 弹出确认对话框
   "确定要删除「激进型」吗？
    此操作不可逆。"
         ↓
4. 用户确认
         ↓
5. 调用删除 API
         ↓
6. 删除成功
   - 从列表中移除
   - 显示成功提示
```

**验收标准**:
- [ ] 删除 default 时有明确错误提示
- [ ] 删除当前激活 Profile 时有明确错误提示
- [ ] 删除前有二次确认
- [ ] 删除后列表自动刷新

#### F7: 导出 Profile (YAML) (P2)

**用户故事**:
> 作为用户，我想导出配置档案为 YAML 文件，用于备份或分享。

**功能描述**:
- 导出为 YAML 格式
- 包含完整的配置项
- 自动下载文件

**导出文件命名**:
```
{profile_name}_config_{timestamp}.yaml
示例：conservative_config_20260403_150000.yaml
```

**YAML 格式示例**:
```yaml
---
profile:
  name: conservative
  description: 保守型配置 - 低仓位，严格止损
  created_at: 2026-04-03T10:00:00Z
  
strategy:
  pinbar:
    min_wick_ratio: 0.6
    max_body_ratio: 0.3
    body_position_tolerance: 0.1
  engulfing:
    max_wick_ratio: 0.4
  ema:
    period: 50
    trend_filter_enabled: true
  mtf:
    enabled: true
    ema_period: 100

risk:
  max_loss_percent: 1.0
  trailing_stop_percent: 1.5
  take_profit_levels:
    - ratio: 0.5
      target_percent: 2.0
    - ratio: 0.3
      target_percent: 4.0

exchange:
  default_leverage: 5
  default_symbols:
    - BTC/USDT:USDT
    - ETH/USDT:USDT
```

**验收标准**:
- [ ] 导出文件格式正确
- [ ] 包含所有配置项
- [ ] 文件自动下载

#### F8: 导入 Profile (YAML) (P2)

**用户故事**:
> 作为用户，我想从 YAML 文件导入配置，快速恢复或迁移。

**功能描述**:
- 上传 YAML 文件
- 解析并验证文件格式
- 预览导入内容
- 选择导入为新的 Profile 或覆盖现有

**导入流程**:
```
1. 用户点击"导入"按钮
         ↓
2. 选择 YAML 文件（或拖拽上传）
         ↓
3. 前端解析并验证格式
         ↓
4. 预览导入内容
   - Profile 名称
   - 配置项数量
   - 配置详情（折叠展示）
         ↓
5. 选择导入模式
   - 创建新 Profile（默认）
   - 覆盖现有 Profile（需确认）
         ↓
6. 调用导入 API
         ↓
7. 导入成功
   - 关闭对话框
   - 刷新列表
   - 显示成功提示
```

**验收标准**:
- [ ] 支持拖拽上传
- [ ] 文件格式验证（非 YAML 格式提示错误）
- [ ] 导入时间 < 2s
- [ ] 导入后配置立即可用

## 七、风险与缓解

### 7.1 技术风险

| 风险 | 影响 | 概率 | 缓解措施 |
|------|------|------|----------|
| 数据库迁移失败 | 高 | 低 | 备份数据库，提供回滚脚本 |
| 切换时配置丢失 | 高 | 低 | 事务保护，失败回滚 |
| YAML 格式不兼容 | 中 | 中 | 严格 Schema 验证 |

### 7.2 用户风险

| 风险 | 影响 | 概率 | 缓解措施 |
|------|------|------|----------|
| 误删 Profile | 高 | 中 | 二次确认，禁止删除激活 |
| 切换后配置错误 | 高 | 低 | 预览差异，热重载验证 |
| 导入恶意配置 | 中 | 低 | 文件验证，参数范围检查 |
