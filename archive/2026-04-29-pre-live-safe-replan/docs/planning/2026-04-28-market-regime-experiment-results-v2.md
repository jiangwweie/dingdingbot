## 1. 修正前问题说明

### A. 反前瞻问题

v1 脚本中 `compute_regime_at_entry` 使用 `kl.timestamp < entry_time` 选择 1d K 线。
**问题**：CCXT 约定 `KlineData.timestamp` 是 **open timestamp**（K 线开盘时间），
1d K 线的 close_time = timestamp + 86400000ms。
当 entry 发生在某日 00:00–24:00 UTC 之间时，`timestamp < entry_time` 会选中当日 K 线，
但当日 K 线尚未收盘，close 价格是实时价格而非最终收盘价。
这构成 **1-bar lookahead bias**。

**回测器自身 MTF 逻辑的正确做法**（`backtester.py:1101`）：
```python
candle_close_time = ts + higher_tf_period_ms
if candle_close_time <= timestamp:  # 只用已收盘 K 线
```

### B. 风险口径问题

v1 脚本使用 `max_total_exposure=2.0, daily_max_trades=50`（research baseline），
而 Sim-1 runtime 使用 `max_total_exposure=1.0, daily_max_trades=10`。
更宽松的 exposure 允许更多并发仓位，可能影响 2023 亏损幅度。
v2 同时跑两种口径以验证结论对风险口径的敏感度。

## 2. 反前瞻修正方法

修正：`close_time = kl.timestamp + 86400000`，只使用 `close_time <= entry_time` 的 K 线。

```python
# v1 (有 lookahead):
if kl.timestamp < entry_timestamp_ms:
    last_closed_idx = i

# v2 (修正后):
close_time = kl.timestamp + 86400000  # DAY_MS
if close_time <= entry_timestamp_ms:
    last_closed_idx = i
```

效果：entry 发生在当日 1h K 线时，不会使用当日 1d K 线的未收盘数据，
而是使用前一日已收盘的 1d K 线。这消除了 1-bar lookahead。

## 7. 最终结论

**最佳 2023 改善**: E2 (+10.5%)
**2024/2025 代价**: 2024 下降 61.2%, 2025 下降 29.4%
**E4 shadow 平均 bear WR**: 36.5%

### 结论 1：EMA250/EMA200 粗 regime gate 改善不足

最佳改善仅 +10.5%，远低于 50% 显著阈值。
regime gate 的主要效果是减少交易次数，而非改善收益质量。

### 结论 2：不支持开启 SHORT 主线

E4 shadow tracking 显示 bear regime 下 LONG 的平均 win rate = 36.5% > 25%。
bear regime 下 LONG 并非完全无效（部分 bear LONG 是盈利的），
SHORT 不太可能是 LONG 的简单镜像，独立研究价值有限。

### 结论 3：下一阶段应转向更精细 regime 定义

粗粒度 EMA250/EMA200 gate 失败的原因：
- 2023 年 bull regime 下的 LONG 同样亏损 → EMA 分型太粗，无法区分趋势质量
- 2024/2025 代价过大 → 过滤掉了太多有效交易

建议的更精细 regime 定义方向：
- EMA slope + ADX 组合（趋势强度 + 方向确认）
- 波动率收缩/扩张状态（VIX-like regime）
- 多因子 regime：EMA 方向 × ADX 强度 × 波动率状态
- 或放弃 regime gate，接受策略有适用边界，转向策略组合研究
