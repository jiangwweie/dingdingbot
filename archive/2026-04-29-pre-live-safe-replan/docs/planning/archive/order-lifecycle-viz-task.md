## 问题背景

### 用户反馈 (2026-04-04)

1. **订单状态显示异常**
   - 系统中订单状态都是"进行中"
   - 实际币安 APP 模拟盘已无活跃委托订单
   - 只有一笔持仓（系统中当前这笔）

2. **开仓价格缺失**
   - 订单链中开仓角色 (ENTRY) 没有显示价格
   - 止盈 (TP)、止损 (SL) 有价格

3. **K 线图展示不完整**
   - 当前只展示 ENTRY 订单标记
   - TP/SL 订单没有渲染到 K 线图

---

## 需求分析

### 核心问题

| 问题 | 根因 | 影响 |
|------|------|------|
| 订单状态不更新 | 缺少对账机制 (Reconciliation) | 用户无法信任系统状态 |
| ENTRY 没有价格 | 市价单未成交时 `average_exec_price` 为 null | 成交后自然解决 |
| K 线图只展示 ENTRY | 前端只渲染了 ENTRY 标记 | 无法查看完整交易生命周期 |

### 业务流程

```
委托开仓 (ENTRY) → 成交 (成交价 + 成交时间)
                    ↓
                限价止盈 (TP1/TP2, 止盈价 + 止盈时间)
                    ↓
                止损 (SL, 止损价 + 止损时间)
```

**目标**: 在 K 线图上完整展示一笔订单的生命周期

### 子任务 D-详细：外部订单手动关联功能 ⭐

**核心洞察**:
> 对账功能只能同步订单基本信息，无法重建订单链关系（signal_id/parent_order_id/order_role）。
> **解决方案**: 提供手动关联功能，由用户主动建立订单与信号的关系。

---

**用户故事**:

> 作为交易员，我在币安 APP 手动操作了一笔交易（开仓→止盈→止损），
> 我希望在系统中将这些订单与对应的信号关联起来，
> 以便在 K 线图上查看完整的订单生命周期。

---

**使用场景**:
1. 用户在币安 APP 手动开仓并设置止盈止损
2. 系统对账后同步了这些订单，但标记为"外部订单"
3. 用户通过关联功能，将订单与系统中的信号建立关系
4. K 线图完整展示订单链（ENTRY→TP1→SL）

---

**数据模型扩展**:

```python
class Order(FinancialModel):
    # ... 现有字段 ...
    
    # 新增：外部订单标记
    is_external: bool = Field(default=False, description="是否外部订单")
    external_source: Optional[str] = Field(None, description="外部来源：binance_manual/unknown")
    
    # 新增：手动关联字段
    manual_linked_signal_id: Optional[str] = Field(None, description="用户手动关联的信号 ID")
    manual_linked_role: Optional[str] = Field(None, description="用户手动指定的订单角色")
    manual_linked_parent_id: Optional[str] = Field(None, description="用户手动指定的父订单 ID")
    manual_linked_at: Optional[int] = Field(None, description="手动关联时间戳")
```

---

**API 设计**:

```python
# 1. 关联订单到信号
POST /api/v3/orders/{order_id}/link
{
  "signal_id": "sig_xxx",           # 必填：关联的信号
  "order_role": "ENTRY|TP1|SL",     # 必填：订单角色
  "parent_order_id": "ord_xxx"      # 可选：父订单 ID（止盈止损单需要）
}

Response: {
  "status": "success",
  "order_id": "ord_xxx",
  "linked_signal_id": "sig_xxx",
  "linked_role": "ENTRY"
}

# 2. 取消关联
DELETE /api/v3/orders/{order_id}/link

Response: {
  "status": "success",
  "order_id": "ord_xxx",
  "message": "关联已取消"
}

# 3. 查询可关联的信号
GET /api/v3/orders/{order_id}/linkable-signals

Response: {
  "signals": [
    {"signal_id": "sig_xxx", "symbol": "BTC/USDT", "direction": "LONG", "created_at": 1234567890}
  ]
}
```

**限制说明**:

⚠️ **手动关联依赖用户记忆和操作，可能不完整或错误**

| 风险 | 说明 | 缓解措施 |
|------|------|----------|
| 用户忘记关联 | 订单仍然是孤立的 | 添加"待关联"筛选器和提醒 |
| 用户关联错误 | 订单链关系错乱 | 提供取消关联功能 |
| 历史订单太多 | 难以选择正确的信号 | 提供时间范围筛选和搜索 |
| 用户不记得角色 | 无法确定 ENTRY/TP/SL | 提供订单详情提示（价格/时间） |

### K 线图标记设计

| 订单角色 | 标记形状 | 颜色 | 位置 | 说明 |
|---------|---------|------|------|------|
| **ENTRY** | 箭头 ↑/↓ | 蓝色 | 涨时下方/跌时上方 | 入场成交价 |
| **TP1/TP2/..**.| 圆圈 ○ | 绿色 | 涨时上方/跌时下方 | 止盈成交价 |
| **SL** | 圆圈 ○ | 红色 | 涨时上方/跌时下方 | 止损成交价 |

### 数据流

```
GET /api/v3/orders/{order_id}/klines?include_chain=true
  ↓
返回 order_chain: [
  { order_role: "ENTRY", average_exec_price: "67000", filled_at: 1234567890, status: "FILLED" },
  { order_role: "TP1", average_exec_price: "68500", filled_at: 1234567900, status: "FILLED" },
  { order_role: "SL", average_exec_price: null, filled_at: null, status: "OPEN" }
]
  ↓
前端筛选 status === "FILLED" 的订单
  ↓
为每个成交订单创建 SeriesMarker
```
