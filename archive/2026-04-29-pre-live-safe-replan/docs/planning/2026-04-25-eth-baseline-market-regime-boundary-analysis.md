## 1. 问题定义

当前基准策略在 `2024 / 2025` 表现可接受，但在 `2023` 明显失效。需要回答的不是“如何强行把 2023 调成盈利”，而是：

1. `2023` 的失败是否来自参数失配，还是来自市场状态不适配？
2. `LONG-only`、`EMA`、`MTF` 三层中，哪一层更可能是主因？
3. `1h` 周期是否天然应当适应震荡市场？
4. 后续分析应优先走 `regime boundary`，还是优先走“换周期 / 换参数 / 换品种”？

---

## 2. 先给结论

当前最合理的结论是：

1. **不需要彻底否定策略本体，但需要否定“这条策略应当全环境成立”的想法。**
2. **`2023` 失效的第一主因更接近 `LONG-only + regime mismatch`，不是简单的 EMA 或 MTF 参数失配。**
3. **`1h` 周期并不天然适配震荡；当前这条 `1h` 策略本质上仍是趋势延续策略，不是震荡均值回归策略。**
4. **后续最值得先分析的是“市场适用边界 / regime filter”，而不是优先去救 2023。**

## 3. 已有证据汇总

### 3.1 2023 年度结果

- `2023` 年度汇总：`PnL = -3582.63 USDT`
- `Win Rate = 25.40%`
- `Sharpe = -1.798`
- `Max DD = 49.19%`

已有文档结论：

- 全年仅 `1 月` 明显盈利，之后长期失效
- `2-7 月` 连续亏损且 `Win Rate = 0%`
- `10-12 月` 略有回暖，但不足以扭转年度结果

### 3.2 延续能力证据

`2023` 相比 `2024 / 2025`：

- `MFE` 明显更低
- `MAE` 没有显著恶化

含义：

- 不是“进场后更容易大幅逆行”
- 而是“进场后向上的后续空间不足”

### 3.3 路径级证据

`2023` 相比 `2024 / 2025`：

- 更容易先到 `-0.5R`，而不是先到 `+0.5R`
- `+1R / +2R / +3.5R` 可达率显著更低

含义：

- 初始方向更差
- 后续可兑现空间明显不足

### 3.4 Exit 证伪结果

已有研究已经证伪：

- `BE=True` 不能救 `2023`
- 降低 `TP2` 不能救 `2023`
- 更防守型 exit 不能救 `2023`

因此：

- `2023` 问题不是“exit 坏了”
- 更接近“市场没给趋势延续空间”

---

## 4. 2023 为什么失效

从市场根本和策略本质看，当前基准策略不是“任意市场都能做”的 Pinbar 策略，而是：

`顺大 -> 逆小 -> 抓趋势延续`

也就是：

1. 大周期先给出偏多方向（`MTF + EMA`）
2. 小周期出现逆势回撤后的反转形态（`Pinbar`）
3. 假设回撤结束后，市场会继续给出可兑现的多头延续空间

`2023` 的问题在于：

- 并不是每次都没有局部反弹
- 而是这些局部反弹 **更像 bounce，不像可持续 follow-through**

所以它失败的位置更接近：

`回撤结束后，后续延续段不够`

而不是：

- Pinbar 完全无效
- 止盈止损逻辑完全无效
- 单个参数完全写错

## 5. 归因排序

### 5.1 第一主因：`LONG-only` 与市场状态不匹配

这是当前最强归因。

原因：

- 基准策略只做多
- `2023` 更接近熊市反弹 + 震荡结构
- 在这种环境下，多头局部反转未必能演化成趋势延续

换句话说：

不是“看到了反转形态就该做多”，而是“只有在市场给得出多头 follow-through 的时候，LONG-only 才成立”。

### 5.2 第二主因：策略本体是趋势延续，不是震荡适配

`1h` 只是触发节奏，不等于震荡适配。

当前 `1h` 基准的本体仍然是：

- 服从大级别方向
- 小级别等回撤结束
- 用 TP 结构吃延续

因此它天然更依赖：

- 趋势连续性
- follow-through
- 右侧空间

而不是依赖：

- 高频均值回归
- 震荡区间的来回波动

### 5.3 第三主因：`EMA` 会影响表现，但不是根因

已有证据显示：

- `ema_period` 是高敏感参数
- 但 `2023` 中即使更优 EMA 也仍然亏损
- `ema=60` 优于 `ema=120`，但二者都没让 `2023` 变成适配年

因此：

- `EMA` 决定“亏多少 / 信号质量差多少”
- 但不决定“策略在该 regime 下是否成立”

### 5.4 第四主因：`MTF` 之前有真源问题，但不是当前主解释

需要分清两件事：

1. 旧 MTF 语义确实一度污染过研究链
2. 修正后，`2023` 失效仍然主要指向 `regime mismatch`

因此当前不宜把 `2023` 问题简单归咎为：

- `MTF 写错了`
- `MTF 过滤太严`

MTF 更像：

- 研究前提中必须被校正的一层
- 但不是解释 `2023` 失效的第一主因

## 6. 不应该怎样理解 2023

### 6.1 不应理解成“策略彻底错了”

更准确的说法是：

- 策略不是全环境策略
- 有明确适用边界

### 6.2 不应理解成“1h 周期本应自然适应震荡”

当前策略的 `1h` 不是震荡 alpha，而是趋势延续执行节奏。

### 6.3 不应理解成“再调 exit 就能救活”

已有证据已经说明：

- 防守型 exit 不能把 `2023` 救回来

### 6.4 不应理解成“只要换个 EMA 就能修复”

EMA 有影响，但不能逆转市场结构问题。

## 7. 可以继续发散验证的方向

### 7.1 方向 A：做 `regime boundary` 分析

这是当前最推荐的方向。

重点不是立刻加新过滤器，而是先定义：

- 这条策略的“好市场”长什么样
- 这条策略的“坏市场”长什么样

可观察维度：

- higher timeframe EMA slope
- 趋势强度
- 趋势连续性
- 回撤后 follow-through 强度
- 波动扩张 / 收敛状态

### 7.2 方向 B：检验 `Pinbar` 抓到的是“真延续”还是“弱反弹”

这个问题很关键。

若 `2023` 的 Pinbar 更多只对应：

- 短 bounce
- 而非延续起点

那么问题就不是参数，而是：

- 当前“逆小”定义还不够区分“弱修复”和“真恢复”

### 7.3 方向 C：周期迁移验证，但不把它当救命药

`15m` 可以测，但它不该被当成“拯救 2023”的优先方案。

正确问题是：

- 更细周期是在更早捕捉趋势恢复
- 还是在放大噪音和摩擦

当前弱先验更偏向后者，因此周期迁移应视为：

- **跨尺度验证题**
- 而不是 **立即救火题**

### 7.4 方向 D：区分研究候选与 runtime 假设

当前窗口只适合：

- 明确边界
- 形成候选假说
- 给出后续研究方向

不适合：

- 直接把新想法灌进当前 `sim1_eth_runtime`

## 8. 本窗口建议分析主题

当前最适合在本窗口继续做的，是：

### 主题：ETH 基准策略的市场适用边界分析

可拆成 3 个子问题：

1. 这条策略在什么市场结构下成立？
2. `2023` 失效的主机制是“方向不对”还是“后续空间不足”？
3. 下一步更值得补的是 `regime filter`，还是周期/参数迁移验证？

---

## 9. 当前阶段不建议做的事

1. 不建议为了“救 2023”直接改 runtime 基准
2. 不建议把 `15m` 当作默认解法
3. 不建议再次优先回到 exit 微调
4. 不建议把单年回测改进直接升级成主线结论
5. 不建议让研究链反向污染当前 Sim-1 冻结 runtime

---

## 10. 结论摘要

一句话总结：

**2023 并不是在否定 Pinbar 本身，而是在否定“只靠 LONG-only + 顺大逆小定义，就足以保证后续延续空间”这个假设。**

因此当前更值得做的，不是强行让 `2023` 赚钱，而是明确：

- 哪些市场状态下这条策略该开
- 哪些市场状态下它本来就不该开

这会比继续调 `EMA / MTF / 周期` 更贴近策略本质，也更符合当前 Sim-1 阶段“研究链不反向污染 runtime”的边界。
