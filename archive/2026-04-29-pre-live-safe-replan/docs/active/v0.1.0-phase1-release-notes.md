## 新增功能

### 后端实现

#### 1. 递归 LogicNode 类型系统 (`src/domain/logic_tree.py`)

- 支持 AND/OR/NOT 逻辑门
- 支持 Trigger 和 Filter 叶子节点
- 使用 Pydantic Discriminator Union 实现类型安全
- 限制递归深度 ≤ 3

```python
# 示例：Pinbar + EMA + MTF 组合策略
logic_tree = {
    "gate": "AND",
    "children": [
        {
            "gate": "OR",
            "children": [
                {"type": "trigger", "id": "t1", "config": pinbar_config},
                {"type": "trigger", "id": "t2", "config": "engulfing_config"}
            ]
        },
        {"type": "filter", "id": "f1", "config": ema_config},
        {"type": "filter", "id": "f2", "config": mtf_config}
    ]
}
```

**Git 提交**: `098eb68`

---

#### 2. 递归评估引擎 (`src/domain/recursive_engine.py`)

- `evaluate_node()` 递归函数实现
- AND 节点：`all()` 短路判定
- OR 节点：`any()` 短路判定
- NOT 节点：结果反转
- 返回 TraceNode 树记录完整评估路径

**核心 API**:
```python
from src.domain.recursive_engine import evaluate_node, TraceNode

trace_tree = evaluate_node(
    node=logit_tree,
    kline=kline_data,
    context=filter_context,
    runner=strategy_runner
)
```

**Git 提交**: `b0ec547`

---

#### 3. StrategyDefinition 升级 (`src/domain/models.py`)

- 新增 `logic_tree` 字段支持递归配置
- 向后兼容平铺模式（trigger + filters）
- 自动迁移验证器

**数据模型变更**:
```python
class StrategyDefinition(BaseModel):
    id: str
    name: str
    logic_tree: Optional[LogicNode] = None  # 新增字段
    # 向后兼容字段
    trigger: Optional[TriggerConfig] = None
    filters: List[FilterConfig] = []
    filter_logic: Literal["AND", "OR"] = "AND"
```

**Git 提交**: `838892f`

---

#### 4. 热预览接口 (`POST /api/strategies/preview`)

- 不持久化、不热重载
- 临时拉取最新 K 线数据
- 返回完整 Trace 树评估路径

**请求示例**:
```bash
curl -X POST http://localhost:8000/api/strategies/preview \
  -H "Content-Type: application/json" \
  -d '{
    "logic_tree": {...},
    "symbol": "BTC/USDT:USDT",
    "timeframe": "15m"
  }'
```

**响应示例**:
```json
{
  "signal_fired": true,
  "trace_tree": {
    "node_id": "root",
    "node_type": "gate",
    "gate_type": "AND",
    "passed": true,
    "children": [...]
  },
  "details": {
    "trigger": {...},
    "kline": {...}
  }
}
```

**Git 提交**: `6943b80`

#### 2. 递归渲染组件

**NodeRenderer.tsx** - 递归渲染器
- 根据节点类型自动委托到对应组件
- 支持展开/折叠

**LogicGateControl.tsx** - 逻辑门控制组件
- AND=蓝色、OR=绿色、NOT=红色
- 虚线边框视觉层次
- 子节点管理（添加/删除/移动）

**LeafNodeForm.tsx** - 叶子节点表单
- Schema 驱动参数输入
- Trigger 类型：Pinbar, Engulfing, Doji, Hammer
- Filter 类型：EMA, MTF, ATR, Volume 等

**Git 提交**: `8c2f6d7`

#### 3. 热预览交互 UI

**TraceTreeViewer.tsx** - Trace 树可视化组件
- 通过=绿色/失败=红色 状态标记
- 递归展开/折叠
- 层级深度显示

**StrategyWorkbench.tsx** - 策略工作台增强
- "立即测试"按钮
- 币种/周期选择器
- 评估结果树展示

**api.ts** - 预览 API 调用

## 技术债务与已知问题

### TraceEvent 字段命名不一致 ⚠️

**问题**: 后端 `filter_name` vs 前端 `stage`

| 后端字段 | 前端字段 | 建议 |
|----------|----------|------|
| `filter_name` | `stage` | 统一为 `node_name` |
| `context_data` | `details` | 统一为 `metadata` |

**解决方案**: 第二阶段统一字段命名

旧策略格式自动迁移到递归树：

```python
  "filters": [...],
  "filter_logic": "AND"
}

# 自动转换为
{
  "logic_tree": {
    "gate": "AND",
    "children": [
      {"type": "trigger", ...},
      ...filters
    ]
  }
}
```
