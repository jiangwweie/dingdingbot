## 新功能

### 1. 配置快照版本化（Rollback）

**问题**: 用户在调整策略配置后无法快速回滚到之前的版本，误操作可能导致实盘策略失效。

**解决方案**: 实现配置快照的完整 CRUD 功能，支持一键回滚。

**核心功能**:

- **保存配置快照** - 将当前配置保存为带版本号的快照
- **查看历史版本** - 分页查询所有历史快照
- **一键回滚** - 激活任意历史快照（自动停用当前快照）
- **删除快照** - 清理不需要的历史版本

**数据模型**:

```python
class ConfigSnapshot(BaseModel):
    id: int
    version: str                    # 版本号，如 "v1.0.0"
    config_json: str                # 序列化的配置 JSON
    description: str                # 快照描述
    created_at: str                 # ISO 8601 时间戳
    created_by: str                 # 创建者
    is_active: bool                 # 是否为当前活跃快照
```

**API 端点**:

| 端点 | 方法 | 说明 |
|------|------|------|
| `/api/config/snapshots` | POST | 创建快照 |
| `/api/config/snapshots` | GET | 列表查询 |
| `/api/config/snapshots/{id}` | GET | 获取详情 |
| `/api/config/snapshots/{id}/activate` | POST | 激活/回滚 |
| `/api/config/snapshots/{id}` | DELETE | 删除快照 |

### 2. 异步 I/O 队列优化

**问题**: 高并发场景下，信号尝试落盘可能阻塞主流程，且缺乏背压监控和异常恢复机制。

**解决方案**: 优化异步队列，增加配置化、背压监控和自动恢复。

**核心功能**:

- **队列参数配置化** - 从 `core.yaml` 读取 `batch_size`、`flush_interval`、`max_queue_size`
- **背压监控告警** - 队列超过 80% 容量时输出警告日志
- **连续错误追踪** - 连续 3 次错误时输出严重错误日志
- **自动恢复机制** - Worker 异常退出后自动重启，指数退避（最多 5 次）

**配置示例**:

```yaml
# config/core.yaml
signal_pipeline:
  cooldown_seconds: 14400
  queue:
    batch_size: 10             # 批量落盘大小
    flush_interval: 5.0        # 最大等待时间 (秒)
    max_queue_size: 1000       # 队列最大容量
```

**背压告警日志**:

```
[WARNING] BACKPRESSURE ALERT: Queue size (850) approaching max capacity (1000)
[ERROR] CRITICAL: 3 consecutive flush worker errors, potential database connectivity issue
```

### 3. 指标计算缓存

**问题**: 多策略运行时，每个策略独立创建 EMA 实例，导致重复计算和内存浪费。

**解决方案**: 实现共享 EMA 缓存层，多策略共用同一指标实例。

**核心功能**:

- **共享缓存** - 相同 `symbol:timeframe:period` 返回同一实例
- **TTL 过期** - 默认 1 小时过期，自动清理
- **LRU 淘汰** - 超过容量时淘汰最久未访问的条目
- **并发安全** - 使用 `asyncio.Lock` 保证线程安全
- **统计信息** - 支持查询缓存命中率、大小等指标

**缓存键设计**:

```
键格式："{symbol}:{timeframe}:{period}"
示例："BTC/USDT:15m:60"
```

**使用示例**:

```python
from src.domain.indicators import EMACache

cache = EMACache(ttl_seconds=3600, max_size=1000)

# 获取或创建 EMA 计算器
ema = await cache.get_or_create("BTC/USDT", "15m", 60)

# 更新状态
ema.update(close_price)

# 查询统计
stats = await cache.get_stats()
# {
#   "size": 5,
#   "max_size": 1000,
#   "entries": {
#     "BTC/USDT:15m:60": {
#       "period": 60,
#       "is_ready": True,
#       "access_count": 15,
#       "age_seconds": 120.5
#     }
#   }
# }
```

## 技术亮点

### 1. 配置快照原子性

创建新快照时自动停用所有现有快照，确保任意时刻只有一个活跃快照。

### 2. 背压监控阈值

队列超过 80% 容量时告警，给运维留出响应时间。

### 3. 指数退避恢复

Worker 重启延迟：1s → 2s → 4s → 8s → 16s，避免雪崩。

### 4. 缓存共享设计

多策略共用 EMA 实例，减少内存占用和重复计算。
