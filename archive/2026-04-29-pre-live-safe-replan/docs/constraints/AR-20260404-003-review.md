### 2. Orders 页面 react-window 修复 ⚠️

**修复内容**: react-window 2.x API 适配

**代码证据** (`OrderChainTreeTable.tsx:376-384`):
```tsx
// 已适配 react-window 2.x API
<List
  rowComponent={Row}
  rowProps={{ data: flatData }}
  rowCount={flatData.length}
  rowHeight={52}
  listRef={listRef}
>
  {null}
</List>
```

**Row 组件定义** (`OrderChainTreeTable.tsx:173-175`):
```tsx
const Row = ({ index, style, data }: { ... }): ReactNode => {
  const item = data[index];  // ⚠️ 如果 data 为 undefined 会报错
  if (!item) return null;    // ✅ 有 item 的 null 检查
```

**评审结论**: ⚠️ **需要增强**

**问题**:
- Row 组件缺少对 `data` 本身的 null/undefined 检查
- 如果 `rowProps` 传递失败，`data[index]` 会抛出 TypeError

**建议增强**:
```tsx
const Row = ({ index, style, data }: { ... }): ReactNode => {
  if (!data || !data[index]) return null;  // 双重 null 检查
  const item = data[index];
```

### 3. _get_config_entry_repo fallback 处理 ✅

**修复内容**: 抛出 503 而非返回未初始化实例

**代码证据** (`api.py:263-267`):
```python
def _get_config_entry_repo() -> Any:
    """Get config entry repository or raise error if not initialized."""
    if _config_entry_repo is None:
        raise HTTPException(status_code=503, detail="Config entry repository not initialized. Please restart the server.")
```

**评审结论**: ✅ **正确**
