> **性质**: 基于系统现状反推"隐含目标画像"，不替用户做决策
> **日期**: 2026-04-29
> **阶段**: 系统体检前置文档
> **版本**: v0.1-draft

## 1. 结论摘要

当前系统从代码、配置、策略设计和文档中反推出的目标画像高度一致：**这是一个面向保守型、低频、单币种（ETH）、仅做多、全自动、纯形态识别驱动的合约量化交易系统**。

**核心矛盾点**：
- 回测基准（backtest-parameters.md）默认档的 3 年 Sharpe 仅 0.71，MaxDD 49.19%，并非一个"令人安心"的基准。
- R1b 审计发现只有 risk=0.5% + exposure=1.0 这一组勉强满足 MaxDD<=35%（32.42%），但对应的 PnL 仅 2,113 USDT（3 年累计），年化收益极低。
- 系统在 Sim-1 阶段硬编码了 LONG-only，但方向策略本身在模型层支持 LONG/SHORT，这意味着 SHORT 是"可扩展但未激活"的能力。
- 系统已实现 Engulfing 策略，但 Sim-1 runtime profile 未启用它，说明多策略组合是架构预留但尚未成为实际研究方向。

**适配性判断**：当前系统更适合"接受低频低收益、愿意承受单年最大回撤 35-50%、对自动化有强需求、且以 ETH 为主要关注标的"的交易者。不适合"期望高频、多币种对冲、做空获利、或对回撤有严格约束（<20%）"的交易者。

## 2. 隐含约束推断表

| 约束维度 | 当前系统推断 | 推断依据 | 确认状态 |
|----------|-------------|---------|---------|
| **交易频率** | 低频（日均 0.2-0.5 笔） | 1h 周期 + Pinbar 形态触发 + EMA/MTF 双重过滤 + 4 小时信号冷却期（`cooldown_seconds=14400`），回测 3 年仅 174-203 笔 | 已可从系统推断 |

| **每日盯盘能力要求** | 不需要（全自动） | 信号触发→风控计算→下单→止盈止损全自动执行，通知仅用于信息推送（飞书 webhook） | 已可从系统推断 |

## 4. 系统适配性判断

### 4.1 当前系统更适合的用户/目标

| 特征 | 说明 |
|------|------|
| **低频交易者** | 系统日均信号 0.2-0.5 笔，适合不追求高频的用户 |
| **ETH 看多信仰者** | LONG-only 设计 + ETH 单币种，天然适合"看好 ETH 长期上涨"的用户 |
| **全自动执行偏好者** | 从信号到下单全链路自动，无需人工干预，适合"不想盯盘"的用户 |
| **可承受中高回撤** | 回测 MaxDD 32-49%，适合能接受"账面亏损 30-50%"的用户 |
| **保守风险管理者** | 单笔 1% 风险 + 每日 10% 上限 + 最低余额保护，风控框架完善 |
| **技术驱动型交易者** | 系统高度工程化（Clean Architecture + PG + asyncio），适合有技术背景的用户 |

### 4.2 当前系统不适合的用户/目标

| 特征 | 原因 |
|------|------|
| **高频交易者** | 1h 周期 + 形态过滤，信号密度远不够高频需求 |
| **多币种对冲交易者** | Sim-1 仅支持 1 个币种，多币种需大幅扩展 |
| **做空获利交易者** | Sim-1 硬编码 LONG-only，做空需架构修改 |
| **低回撤要求者（MaxDD<20%）** | 当前最保守配置 MaxDD 仍达 32%，无法满足 |
| **半自动/人工确认交易者** | 无信号审批机制，全自动执行 |
| **复杂策略研究者** | 当前仅 Pinbar + EMA/MTF，无 regime/multi-factor/funding 等高级维度 |
| **跨品种套利交易者** | 无价差追踪、无配对交易逻辑 |

### 4.3 如果用户真有以下目标，系统必须额外改造

| 用户目标 | 所需改造 | 工作量估计 |
|----------|---------|-----------|
| MaxDD < 20% | 降低 risk%、减小 exposure、增加过滤器、或引入动态仓位管理（如 Kelly 公式） | 中-高（需重新研究） |
| 做空（SHORT） | 修改 `StrategyRuntimeConfig` 验证器 + 研究做空策略 + 调整风控参数 + 回测验证 | 中（架构支持，主要是研究工作） |
| 多币种（3-5 个） | 扩展 `MarketRuntimeConfig` + 每币种独立 profile + 多币种信号冲突处理 + 流动性评估 | 高 |
| 半自动模式 | 新增信号审批状态机 + 前端审批 UI + 人工响应超时处理 + 通知升级 | 高 |
| Funding rate 追踪 | 新增 funding rate 数据采集 + 持仓成本核算 + 止盈止损调整 | 中 |
| 连续亏损熔断 | 新增"连续 N 月亏损自动减仓/停止"机制 + 月度统计追踪 | 低-中 |
| Benchmark 对比 | 新增 benchmark 回测模块 + 相对收益计算 + 对比报告 | 低-中 |
