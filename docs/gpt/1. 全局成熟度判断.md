## 4. 最大风控问题：你有单笔风控，但没有账户级风控

- 没有连续亏损降仓。

- 没有强平距离检查。
- 没有保证金充足率检查。
- 没有组合层风险检查。

这对小资金尤其危险。小资金不是怕单笔亏损，真正怕的是：

> 连续错误暴露 + 自动化系统继续开仓 + 没有账户状态机。

所以你下一步风控不应该继续研究 `risk=1% vs 2%`，而应该先补一个 **Account Risk State Machine**：

| 状态      | 条件                             | 动作             |
| --------- | -------------------------------- | ---------------- |
| NORMAL    | 无回撤压力                       | 正常风险         |
| CAUTION   | 当日亏损 > 0.75% 或连续 2 笔亏损 | 新信号减半       |
| DEFENSIVE | 当日亏损 > 1.25% 或回撤 > 5%     | 只允许最高置信度 |
| STOP      | 当日亏损 > 2% 或保护单异常       | 禁止新仓         |
| RECOVERY  | 人工确认后恢复                   | 低风险重启       |

这个比继续调参数更重要。

| 资产组                 | 最大风险预算 | 说明                       |
| ---------------------- | ------------ | -------------------------- |
| Crypto majors          | 40%          | BTC / ETH / SOL            |
| Crypto alts            | 20%          | 高波动，严格限制           |
| Metals                 | 20%          | XAU / XAG                  |
| Equity / ETF perps     | 20%          | NVDA / AAPL / SPY / QQQ 等 |
| Cash / no-trade buffer | 永远保留     | 避免满仓满风险             |

但这不是资金比例，而是**风险预算比例**。

例如当账户允许总开放风险为 1.5% 时：

| 资产组        | 可用风险 |
| ------------- | -------- |
| Crypto majors | 0.60%    |
| Crypto alts   | 0.30%    |
| Metals        | 0.30%    |
| Equity / ETF  | 0.30%    |

这样可以避免所有策略同时变成同一个方向的 risk-on 暴露。

# 6. 做空是必要的，但不能简单镜像做空

你愿意做空，这是非常好的一点。对合约系统来说，**不能做空会严重限制 regime 覆盖**。

但我不建议把现有 LONG Pinbar 直接镜像成 SHORT Pinbar。你之前的研究也显示，SHORT-only 仅在特定年份有效，三年累计远弱于 LONG。

做空应该有专门逻辑：

| 做空类型     | 更合理的条件                                      |
| ------------ | ------------------------------------------------- |
| 趋势下跌延续 | 4h downtrend + 1h weak rebound                    |
| 过热回落     | funding 高 + price 贴近 Donchian high + 动能衰减  |
| 跌破后回踩   | breakdown + failed reclaim                        |
| 美股类合约   | session momentum / gap failure                    |
| 贵金属       | 避险趋势结束后的 volatility contraction breakdown |

做空不是“看见反 Pinbar 就 short”，而是要判断：

> 市场是在回调，还是在趋势反转？

# 7. 全自动的前提：执行链必须先修好

你选择全自动，这会大幅提高系统要求。

对半自动系统，这个问题可以人工补。
对全自动系统，这是 P0 阻塞。

6. 风控拒绝必须结构化落库。
7. 每日 PnL / equity curve 必须可观测。
8. kill switch 必须一键生效。

所以你现在不缺策略想法，缺的是：
