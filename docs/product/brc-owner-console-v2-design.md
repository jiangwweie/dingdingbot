# BRC Owner Console v2 设计文档

## 1. 设计目标

Owner Console v2 的设计目标是把旧的“后端状态说明页”重构为：

```text
Owner 主控工作台
```

它需要让 Owner 在较短时间内完成判断：

```text
当前系统是否安全？
当前账户是否正常？
当前策略组是什么？
当前是否有持仓、挂单、盈亏？
最近有没有执行意图？
当前阻断是什么？
下一步能做什么？
```

控制台不追求展示所有后端字段，而是将系统事实转译成 Owner 能理解的状态、动作和风险。

## 2. 信息架构

### 2.1 一级导航

使用左侧导航。

```text
首页
策略组
执行意图
账户订单
复盘分析
链路追踪
```

### 2.2 页面职责

| 页面     | 角色           | 主要信息                                                   |
| -------- | -------------- | ---------------------------------------------------------- |
| 首页     | 主控面板       | 系统状态、账户状态、当前策略组、当前阻断、最近意图、下一步 |
| 策略组   | 策略工作台     | 策略组、具体策略、状态、信号、意图、下一步                 |
| 执行意图 | 只读意图台账   | 策略想做什么、为什么未执行、后续表现                       |
| 账户订单 | 简化交易所视图 | 权益、余额、保证金、持仓、挂单、盈亏                       |
| 复盘分析 | 证据和结论     | 风险披露、Owner 接受、review、信号/意图证据                |
| 链路追踪 | 可解释链路     | 状态节点、来源、技术详情、raw metadata                     |

## 3. 全局布局

### 3.1 桌面布局

```text
┌────────────────┬──────────────────────────────────────────────┐
│ 策略控制台       │ 顶部栏：● 只读观察中       刷新  用户/退出       │
│                ├──────────────────────────────────────────────┤
│ 首页            │                                              │
│ 策略组          │ 页面内容                                      │
│ 执行意图        │                                              │
│ 账户订单        │                                              │
│ 复盘分析        │                                              │
│ 链路追踪        │                                              │
│                │                                              │
│ 底部：用户/退出  │                                              │
└────────────────┴──────────────────────────────────────────────┘
```

### 3.2 左侧导航

左侧导航应包含：

```text
应用名称：策略控制台
导航项
当前用户 / 退出
```

导航项使用图标 + 中文名称：

| 导航     | 建议图标含义 |
| -------- | ------------ |
| 首页     | 仪表盘       |
| 策略组   | 分支 / 节点  |
| 执行意图 | 箭头 / 闪电  |
| 账户订单 | 钱包 / 列表  |
| 复盘分析 | 图表         |
| 链路追踪 | 流程图       |

不再显示旧英文导航。

### 3.3 顶部状态胶囊

顶部右侧显示全局状态胶囊。

示例：

```text
● 只读观察中
```

或：

```text
● 实盘只读 · 记录意图
```

状态点规则：

| 颜色 | 状态             |
| ---- | ---------------- |
| 绿色 | 安全观察中       |
| 黄色 | 有阻断或等待处理 |
| 红色 | 风险阻断         |
| 灰色 | 未连接 / 未上报  |

鼠标悬停显示短提示：

```text
当前连接真实市场，只记录信号和执行意图。不会创建执行指令，也不会下单。
```

点击后打开小弹层：

```text
当前模式：实盘只读
权限：记录意图
下单：禁止
执行指令：禁止
最近检查：23:51
```

## 4. 首页设计

### 4.1 首页目标

首页是 Owner 每次打开控制台的第一屏。

它必须在一屏内回答：

```text
当前是什么状态？
账户怎么样？
当前策略组是什么？
为什么卡住？
有没有执行意图？
下一步能做什么？
```

### 4.2 首页结构

```text
┌──────────────────────────────────────────────────────────────┐
│ 当前结论                                                       │
│ MI-001 SOL 已完成试验前准备，当前因启动保护未预检而阻断，试验未启动。 │
│ 模式：实盘只读 · 记录意图 · 禁止下单                            │
└──────────────────────────────────────────────────────────────┘

┌──────────────┬──────────────┬──────────────┬──────────────┐
│ 系统状态      │ 账户状态      │ 当前策略组     │ 当前阻断      │
│ 安全          │ 读取正常      │ MI-001 SOL    │ 启动保护未预检 │
│ 禁止下单      │ 持仓 0 / 挂单 0│ 多头观察       │ 试验未启动     │
└──────────────┴──────────────┴──────────────┴──────────────┘

┌──────────────────────────────┬──────────────────────────────┐
│ 账户概览                       │ 最近执行意图                  │
│ 总权益：-- USDT                 │ 暂无新意图                    │
│ 可用余额：-- USDT               │ 最近信号：--                  │
│ 未实现盈亏：--                  │ 最近阻断：启动保护            │
│ 持仓：0                         │                               │
│ 挂单：0                         │                               │
└──────────────────────────────┴──────────────────────────────┘

┌──────────────────────────────────────────────────────────────┐
│ 当前可做                                                       │
│ [查看策略组] [查看证据] [查看链路]                              │
│ 当前不可做：启动试验 / 创建执行指令 / 下单                      │
└──────────────────────────────────────────────────────────────┘
```

### 4.3 首页数据来源

首页可组合以下数据：

```text
readiness
account facts
strategy family registry
MI-001 SOL readiness / registration state
operation capability
execution permission
latest signal / intent evidence
```

如果字段缺失，使用：

```text
暂未上报
暂无数据
未进入该阶段
```

不得伪造。

## 5. 信息输入入口设计

### 5.1 入口位置

首页提供明显入口：

```text
新建观察
```

可以放在首页顶部右侧或“当前可做”区域。

### 5.2 交互方式

采用向导式，不采用对话框优先。

```text
步骤 1：选择市场结构
[趋势延续] [震荡回归] [波动突破] [事件驱动]

步骤 2：选择标的
[SOL] [ETH] [BNB] [BTC]

步骤 3：系统推荐策略组
MI-001 SOL 多头
当前模式：只读观察 / 记录意图
风险提示：高波动，尚未进入下单阶段

步骤 4：确认进入观察
[确认观察] [取消]
```

### 5.3 本期实现范围

v2 首版可以只做入口占位：

```text
新建观察功能稍后开放
```

不要求实现完整推荐逻辑。

### 6.1 页面目标

展示策略组、具体策略和当前状态。

不是展示数据库字段。

### 6.2 策略组列表

│ 策略组                                                        │
│ 当前共有 3 个候选，1 个已进入试验前准备                        │
└──────────────────────────────────────────────────────────────┘

┌──────────────┬────────────┬────────────┬────────────┬────────────┐
│ 策略组        │ 状态        │ 标的        │ 最近信号     │ 下一步       │
├──────────────┼────────────┼────────────┼────────────┼────────────┤
│ MI-001 动量冲击 │ 准备完成    │ SOL 多头     │ 暂无         │ 查看详情     │
│ MI-001 动量冲击 │ 候选        │ BNB 多头     │ 暂无         │ 准备观察     │

```text

├──────────────┼────────────┼────────────┼────────────┤

│ BNB 多头      │ 候选        │ 未开始       │ 无           │
└──────────────┴────────────┴────────────┴────────────┘

```

## 7. 执行意图页面设计

### 7.1 页面目标

展示策略在只读观察中“本来想做什么”。

执行意图不是订单。

### 7.2 列表结构

```text
┌──────────────────────────────────────────────────────────────┐
│ 执行意图                                                       │
│ 这里只记录策略意图，不会触发交易。                              │
└──────────────────────────────────────────────────────────────┘

┌──────────┬──────────────┬──────┬──────┬────────┬────────────┬────────┐
│ 时间      │ 策略          │ 标的  │ 方向  │ 意图    │ 状态        │ 后续表现 │
├──────────┼──────────────┼──────┼──────┼────────┼────────────┼────────┤
│ 10:00    │ MI-001 SOL    │ SOL  │ 多   │ 想入场  │ 已记录未执行 │ +1.2%  │
│ 14:00    │ MI-001 SOL    │ SOL  │ 多   │ 想入场  │ 被阻断       │ -0.8%  │
└──────────┴──────────────┴──────┴──────┴────────┴────────────┴────────┘
```

### 7.3 详情设计

点击一条意图展示：

```text
为什么出现这个意图
当时价格
当时账户状态
为什么没执行
后续 1h / 3h / 24h 表现
```

后续可接 TradingView 类图表，在图上标记信号点和意图点。

本期只预留位置，不实现图表。

## 10. 链路追踪页面设计

### 10.1 页面目标

解释一件事如何走到当前状态。

它替代旧的“审计详情”。

### 10.2 流程图结构

采用纵向流程线。

```text
● 策略候选形成        已完成
│  MI-001 SOL long
│
● Owner 风险接受      已完成
│  已接受进入准备阶段
│
● PG 注册             已完成
│  已写入主数据源
│
● 最终复核            已完成
│  final pre-start review
│
● 启动保护            阻断
   需要运行时预检
```

### 10.3 节点展开

每个节点可展开：

```text
状态说明
来源
时间
关联对象
技术字段
raw JSON
```

技术字段只在展开后显示。

## 12. 数据适配策略

Codex 可自由组合现有 API：

```text
readiness
account facts
strategy families
admission decisions
trial bindings
operation detail
review / evidence
campaign metadata
trial_trade_intent evidence
```

如果 API 不存在或字段缺失：

```text
显示暂未上报
显示暂无数据
显示未进入该阶段
```

不得伪造数据。

## 13. 旧页面处理

旧页面不再作为产品入口。

处理策略：

```text
移除导航
移除路由入口
旧路由跳转到 /home 或显示已下线
旧文件可保留但不引用
```

不要求首版物理删除旧文件。

## 14. 验收标准

### 14.1 导航

主导航只出现：

```text
首页
策略组
执行意图
账户订单
复盘分析
链路追踪
```

### 14.2 首页

一屏能看懂：

```text
系统状态
账户状态
当前策略组
当前阻断
最近意图
下一步
```

### 14.3 技术字段

技术字段不应抢占主界面。

必须默认折叠或放入链路追踪页。

### 14.4 安全

全站不得出现启用状态的：

```text
Start Trial
Place Order
Enable Live
Execute Order
Create Execution Intent
Withdraw
Transfer
Flatten
Cancel Order
```

### 14.5 截图验收

必须保存：

```text
home.png
strategy-groups.png
intents.png
account-orders.png
analysis.png
trace.png
legacy-route-redirect.png
```
