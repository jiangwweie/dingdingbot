# BRC Owner Console v2 UI 视觉规范

## 1. 目标

Owner Console v2 的 UI 目标是把控制台从“文档式状态页”改为“Owner 主控工作台”。

界面需要让 Owner 一眼看懂：

```text
系统是否安全
账户是否正常
当前策略组是什么
最近有没有执行意图
哪里被阻断
下一步能做什么
```

界面不应让 Owner 感觉自己在阅读技术报告、Markdown 文档或后端调试面板。

## 2. 视觉定位

推荐视觉方向：

```text
现代交易工作台
克制的 SaaS Dashboard
类似 Linear / Raycast / Vercel Dashboard 的清爽风格
带少量交易所工作台感
```

避免：

```text
传统后台管理系统
文档页面
Markdown 报告页
密集表格系统
开发调试面板
```

## 3. 主题模式

前端必须支持：

```text
浅色模式
深色模式
跟随系统
```

建议默认优先适配深色模式，但浅色模式也必须可用。

### 3.1 深色模式原则

深色模式适合长期观察策略、账户和市场状态。

深色模式应具备：

```text
深色背景
低亮度边框
柔和卡片
状态色清晰
字体对比充分
少量高亮
```

不应使用：

```text
纯黑大面积背景
高饱和霓虹色
过多发光效果
复杂渐变
```

### 3.2 浅色模式原则

浅色模式应保持干净、轻量。

浅色模式应具备：

```text
浅灰背景
白色卡片
轻边框
柔和阴影
清晰状态色
```

## 4. 布局规范

### 4.1 总体布局

采用：

```text
左侧导航
顶部状态栏
主内容区
```

结构：

```text
┌──────────────┬────────────────────────────────────────┐
│ 左侧导航       │ 顶部状态栏                              │
│              ├────────────────────────────────────────┤
│              │ 主内容区                                │
└──────────────┴────────────────────────────────────────┘
```

不使用顶部横向主导航作为主要导航方式。

### 4.2 左侧导航

左侧导航包含：

```text
策略控制台
首页
策略组
执行意图
账户订单
复盘分析
链路追踪
用户 / 退出
```

导航样式：

```text
图标 + 中文名称
当前页面高亮
间距充足
不要太窄
不要堆太多副信息
```

### 4.3 顶部状态栏

顶部状态栏只承载全局运行状态，不做复杂导航。

包含：

```text
全局状态胶囊
刷新
用户状态
```

示例：

```text
● 只读观察中
```

或：

```text
● 实盘只读 · 记录意图
```

## 5. 全局状态胶囊

全局状态胶囊是 v2 的关键组件。

### 5.1 展示形式

状态胶囊应像连接状态按钮：

```text
● 只读观察中
```

或：

```text
● 实盘只读 · 记录意图
```

要求：

```text
圆点 + 短文字
胶囊形状
轻边框
可悬停
可点击打开详情
```

### 5.2 状态颜色

| 状态            | 颜色语义     |
| --------------- | ------------ |
| 安全 / 正常     | 绿色         |
| 只读观察        | 青色或蓝绿色 |
| 等待 / 阻断     | 黄色         |
| 风险 / 禁止     | 红色         |
| 未连接 / 未上报 | 灰色         |

### 5.3 悬停提示

鼠标悬停时显示短提示，不超过两行。

示例：

```text
当前连接真实市场，只记录信号和执行意图。
不会创建执行指令，也不会下单。
```

### 5.4 点击弹层

点击状态胶囊后显示小弹层：

```text
当前模式：实盘只读
权限：记录意图
下单：禁止
执行指令：禁止
最近检查：23:51
```

弹层不能变成大段说明。

## 6. 信息密度

### 6.1 一屏优先

首页必须尽量一屏看清。

不允许：

```text
长篇说明
密集小字
多层嵌套卡片
大量 raw 字段
长 checklist 平铺
```

### 6.2 卡片数量

首页第一屏建议不超过：

```text
1 个当前结论区
4 个状态卡
2 个内容卡
1 个当前可做区
```

### 6.3 每张卡片字段数量

普通卡片最多展示：

```text
3-5 个核心字段
```

更多字段进入详情。

## 7. 文案规范

### 7.1 中文优先

主文案使用简体中文。

允许保留必要英文：

```text
SOL
ETH
BTC
USDT
API
PG
TradingView
execution intent
```

不推荐在主界面直接显示长英文枚举。

### 7.2 主文案规则

主文案应是 Owner 能理解的状态句。

推荐：

```text
运行时启动保护还没有完成预检，试验暂不能启动。
```

不推荐：

```text
blocked_startup_guard_runtime_coupled
```

技术字段可以作为标签：

```text
技术状态：blocked_startup_guard_runtime_coupled
```

### 7.3 短句优先

每段说明控制在 1-2 行。

避免：

```text
本页面用于展示……
该字段表示……
由于系统当前……
```

这类文档式说明。

### 7.4 “不是”类说明要集中

安全边界不要在每个卡片反复写。

推荐集中展示：

```text
当前模式：只读观察
禁止：下单、执行指令、启动试验
```

避免每个页面重复长句：

```text
这不是订单，这不是执行意图，这不会……
```

## 8. 技术详情规范

### 8.1 默认折叠

所有技术字段默认折叠。

统一入口：

```text
技术详情
```

默认状态：

```text
技术详情 ▸
```

展开状态：

```text
技术详情 ▾
```

### 8.2 技术详情内容

可展示：

```text
terminal_state
runtime_status
strategy_family_id
version_id
source_refs
operation_id
preflight_id
raw JSON
API fallback
```

### 8.3 技术详情位置

技术详情不应放在首页第一视觉中心。

可以放在：

```text
卡片底部
详情抽屉
链路追踪页
```

## 9. 卡片规范

### 9.1 普通状态卡

结构：

```text
标题
主值
副说明
状态点 / 状态胶囊
```

示例：

```text
系统状态
安全
只读观察中，禁止下单
```

### 9.2 账户指标卡

结构：

```text
指标名
数值
单位
变化 / 状态
```

示例：

```text
可用余额
-- USDT
暂未上报
```

### 9.3 策略卡

结构：

```text
策略组名称
具体策略
状态
标的 / 方向
下一步
```

示例：

```text
MI-001 动量冲击
SOL 多头观察
试验前准备完成
当前阻断：启动保护未预检
```

## 10. 表格规范

表格用于列表，不用于展示所有字段。

### 10.1 策略组表格

字段建议：

```text
策略组
具体策略
标的
方向
状态
最近信号
下一步
```

### 10.2 执行意图表格

字段建议：

```text
时间
策略
标的
方向
意图
状态
原因
后续表现
```

### 10.3 表格视觉

要求：

```text
行高充足
状态用胶囊
不要小字挤压
不要超过 7-8 列
必要时用详情抽屉
```

## 11. 空态规范

空态不能像错误。

### 11.1 Campaign / 试验未启动空态

推荐：

```text
当前没有已启动试验。
MI-001 SOL 仍处于试验前准备阶段，因此没有可复盘的试验记录。
```

不推荐：

```text
暂无 campaign
数据库未返回 latest campaign
```

### 11.2 执行意图空态

推荐：

```text
暂无执行意图。
策略还没有产生可记录的意图，或当前仍处于准备阶段。
```

### 11.3 账户数据缺失

推荐：

```text
暂未上报
```

不推荐：

```text
undefined
null
not_available
```

技术值可放技术详情。

## 12. 链路追踪视觉规范

“链路追踪”用于展示可解释过程。

### 12.1 推荐形式

使用纵向流程线。

```text
● 策略候选形成        已完成
│
● Owner 风险接受      已完成
│
● PG 注册             已完成
│
● 最终复核            已完成
│
● 启动保护            阻断
```

### 12.2 节点颜色

| 节点状态      | 颜色 |
| ------------- | ---- |
| 已完成        | 绿色 |
| 当前阻断      | 黄色 |
| 失败          | 红色 |
| 未开始        | 灰色 |
| 跳过 / 不适用 | 灰蓝 |

### 12.3 节点内容

每个节点默认只显示：

```text
标题
状态
一句说明
```

详细技术字段放展开项。

## 13. 执行意图详情规范

执行意图详情应像复盘卡，而不是订单详情。

内容：

```text
为什么出现这个意图
当时价格
当时账户状态
为什么没执行
后续 1h / 3h / 24h 表现
```

后续可以加图表：

```text
TradingView 类图表
信号点标记
意图点标记
后续走势
```

首版仅预留，不实现。

## 14. 图标规范

可使用 lucide-react 等现有图标库。

建议：

| 页面     | 图标语义             |
| -------- | -------------------- |
| 首页     | LayoutDashboard      |
| 策略组   | GitBranch / Network  |
| 执行意图 | Zap / ArrowUpRight   |
| 账户订单 | Wallet / List        |
| 复盘分析 | LineChart            |
| 链路追踪 | GitCommit / Workflow |

图标只做辅助，不要过度装饰。

## 15. 交互规范

### 15.1 可点击元素

按钮必须明确：

```text
查看策略组
查看复盘
查看链路
技术详情
刷新
```

当前版本禁止出现：

```text
启动试验
下单
启用实盘
创建执行指令
```

### 15.2 Hover

Hover 可用于：

```text
状态胶囊解释
指标解释
技术标签解释
```

Tooltip 应短，不超过两行。

### 15.3 Drawer / 抽屉

用于：

```text
执行意图详情
策略详情
技术详情
```

不要用弹窗展示大段文档。

## 16. 危险按钮规范

全站不得出现 enabled 状态的：

```text
Start Trial
Place Order
Enable Live
Execute Order
Create Execution Intent
Withdraw
Transfer
Flatten
Cancel Order
Start Runtime
```

如果未来展示这些动作，必须：

```text
禁用
标注需要单独授权
不在 v2 首版启用
```

## 17. 响应式规范

首版优先桌面端。

最低要求：

```text
1440px 宽度体验良好
1280px 不崩
移动端可暂缓
```

不要求手机端完整体验。

## 18. 实现约束

优先使用现有：

```text
React
React Router
Tailwind
lucide-react
现有 API client
现有认证
```

不建议新增大型 UI 库。

如果需要新增小工具组件，可以在前端组件目录中实现。

## 19. 验收标准

### 19.1 视觉验收

Owner 打开页面后应感觉：

```text
这是一个主控工作台
不是一篇文档
不是后端状态调试页
```

### 19.2 页面验收

必须访问：

```text
/home
/strategy-groups
/intents
/account-orders
/analysis
/trace
```

### 19.3 截图验收

必须生成：

```text
home.png
strategy-groups.png
intents.png
account-orders.png
analysis.png
trace.png
legacy-route-redirect.png
```

### 19.4 安全验收

全站无可点击危险交易按钮。

### 19.5 文案验收

主文案中文为主。

技术字段不作为主标题。

## 20. 不做事项

首版不做：

```text
完整 TradingView 图表
高级动画
复杂主题编辑器
移动端完整适配
Feishu
LLM selector
交易入口
订单入口
启动试验入口
```

首版只做：

```text
可用、清爽、中文、低密度、可验收的 Owner 主控工作台。
```
