# 交易控制台产品说明书 v0.1

## 0. 文档状态

- 产品名：交易控制台
- 阶段：Gate 2 `PASS_WITH_CONSTRAINT`
- 适用范围：只读版交易控制台 + Gate 3 操作组件预设
- 唯一事实源：`/api/trading-console/*`
- 禁止事实源：
  - `/api/brc/*`
  - `/api/runtime/*`
  - `/api/dev/testnet/brc/*`
- 本阶段不开放真实 action API
- 本阶段不接入 execute / cancel / replace / flatten / retry protection / runtime start / auto-execution

------

## 1. 产品定位

交易控制台是一个真实环境下的 Owner 交易控制界面。

它不是：

- 测试页面
- 模拟页面
- demo 页面
- 普通交易终端
- 策略大后台
- 任意下单页面
- testnet 操作台

它要解决的问题是：

```text
Owner 打开控制台后，能够快速判断：
1. 当前真实账户是否安全
2. 当前是否有真实仓位 / 挂单 / 保护单 / 异常敞口
3. 当前是否有未完成的实盘行动
4. 当前是否存在有界实盘授权
5. 当前执行链路是否存在阻断、warning、unknown、unavailable
6. 当前唯一应该处理的事项是什么
7. 哪些动作现在不可用，为什么不可用
8. 哪些事实来自后端，哪些字段当前不可用
```

交易控制台的核心不是“发起实盘行动”，而是：

```text
真实账户风险状态
+ 订单事实
+ 授权状态
+ 执行链路可见性
+ 保护 / 异常状态
+ 复盘
+ 技术审计
```

------

## 2. Gate 2 产品边界

Gate 2 允许前端做：

- 交易控制台正式产品文档
- API 接入文档
- 页面设计规范
- AI Studio 只读版前端工程规划
- 只读版前端工程
- Gate 3 操作组件预设
- disabled / unavailable / future placeholder 状态展示

Gate 2 不允许前端做：

- 真实执行按钮
- 撤单按钮
- 改单按钮
- 平仓按钮
- retry protection 按钮
- runtime start 按钮
- auto-execution 开关
- 前端自行组合交易动作
- 前端调用旧 action API
- 前端根据多个旧接口自行推断安全状态
- 前端估算 `not_available` 字段
- 前端把 `not_live_connected` / `unknown` / `unavailable` 展示为 clean

## 3. 核心用户目标

### 3.1 Owner 打开首页时

Owner 需要知道：

```text
当前账户有没有真实风险？
是否有仓位？
是否有挂单？
是否有保护单？
是否存在 PG / 交易所不一致？
是否存在无法归属订单？
是否存在未完成实盘行动？
是否存在未消费授权？
当前数据是否连接真实交易所事实？
当前是否有必须处理的异常？
```

### 3.2 Owner 查看订单时

Owner 需要知道：

```text
哪些是交易所订单？
哪些是本地 PG 订单？
哪些订单已匹配？
哪些订单只在 PG 存在？
哪些订单只在交易所存在？
哪些订单状态不一致？
哪些是建仓单？
哪些是 TP / SL 保护单？
哪些是孤儿保护单？
哪些订单属于当前实盘行动？
哪些订单属于历史实盘行动？
哪些订单无法归属？
```

### 3.3 Owner 查看执行控制时

Owner 需要知道：

```text
当前有没有可执行授权？
授权是否已消费？
授权是否过期？
授权是否可行动？
final gate 当前状态是什么？
哪些 gate 通过？
哪些 gate warning？
哪些 gate block？
当前是否存在 deferred execute endpoint？
为什么不能执行？
```

Gate 2 阶段不展示可点击执行按钮。

### 3.4 Owner 查看异常恢复时

Owner 需要知道：

```text
为什么当前状态异常？
阻断的是授权、执行，还是新实盘行动？
是否存在未保护仓位？
是否存在残留挂单？
是否存在孤儿保护单？
是否存在 recovery task？
哪些动作当前只是 future placeholder？
哪些动作需要主控 / 后端安全 action API？
```

Gate 2 阶段不展示可点击的 cancel / flatten / retry protection 操作。

### 3.5 Owner 查看复盘时

Owner 需要同时看到：

```text
交易结果
系统链路结果
异常 / 恢复结果
字段不可用状态
```

复盘不只看盈亏，也要看系统链路是否可靠。

### 3.6 Owner 查看技术审计时

Owner 需要按关键 ID 追链：

```text
authorization_id
intent_id
order_id
exchange_order_id
symbol
```

并查看：

```text
authorization
intents
orders
positions
reviews
audit_events
raw_payload_policy
```

## 6. 全局状态语义

### 6.1 freshness_status

所有页面必须正确展示 freshness。

| 状态                 | 产品含义                             | UI 规则                                  |
| -------------------- | ------------------------------------ | ---------------------------------------- |
| `fresh`              | 请求的数据源可用，且无明显不可用事实 | 可正常展示，但仍需看 warnings / blockers |
| `warning`            | 数据源可用，但存在 warning facts     | 不得展示为完全 clean                     |
| `degraded`           | 一个或多个数据源失败或不可用         | 显著警示                                 |
| `not_live_connected` | 默认未请求真实交易所只读事实         | 不得展示为“账户安全”                     |

### 6.2 unavailable

`unavailable` 表示后端数据源、repository、table、service 或外部事实不可用。

前端规则：

```text
unavailable 不能展示为 clean
unavailable 不能被忽略
unavailable 不能由前端估算
unavailable 必须显式提示数据缺失
```

### 6.3 not_available

`not_available` 表示该字段当前后端没有事实。

典型字段：

- client_order_id
- fees
- fee_asset
- funding
- slippage
- fills_table

前端规则：

```text
not_available 只能展示为不可用
不得估算
不得补默认值
不得隐藏后假装完整
```

### 6.4 warnings

`warnings` 是风险提示，不一定阻断 read model 生成。

前端规则：

```text
warning 必须可见
warning 不等于 hard blocker
warning 不得展示为 clean
```

### 6.5 blockers

`blockers` 是阻断项。

前端规则：

```text
blocker 必须显著展示
blocker 必须说明影响哪个流程
blocker 不得被折叠到技术详情里
```

------

## 7. 全局动作规则

Gate 2 阶段所有真实动作都不开放。

### 7.1 禁止开放的动作

- execute
- cancel order
- replace order
- flatten position
- retry protection
- resolve recovery task
- retry recovery task
- runtime start
- auto-execution enablement
- credential / API-key update
- PG mutation action

### 7.2 deferred action 展示规则

以下字段只能展示为 disabled / unavailable / future placeholder：

- `deferred_actions`
- `future_action_slots`
- `deferred_execute_endpoint`

前端展示文案建议：

```text
后端暂未开放此动作
需要主控确认后端安全 action API
当前仅展示状态，不执行操作
```

### 7.3 Gate 3 操作组件预设

AI Studio 可以预设：

- 操作按钮槽位
- 危险动作按钮样式
- disabled 状态
- unavailable 状态
- future placeholder 状态
- 二次确认弹窗布局
- Recovery 动作区布局
- 订单操作菜单布局
- 执行控制动作区布局

但所有操作组件必须默认不可用。

## 8. 页面说明

# 8.1 首页 / 真实风险总览

### 页面目的

首页不是“发起实盘行动”页面。
首页是 Owner 打开交易控制台后的真实风险总览。

### 页面要回答

```text
当前账户是否有真实风险？
是否连接真实交易所事实？
是否有仓位？
是否有挂单？
是否有保护单？
是否有未完成意图？
是否有有界实盘授权？
是否有 warning / blocker / unavailable？
当前唯一待处理事项是什么？
```

### 页面模块

1. 全局环境条
2. 真实风险状态卡
3. 账户摘要
4. 仓位摘要
5. 挂单摘要
6. 授权摘要
7. 执行控制摘要
8. warning / blocker / unavailable 区
9. 最近复盘摘要
10. 页面入口区

### 主要接口

```
GET /api/trading-console/dashboard-state
```

### 展示规则

- `not_live_connected` 时不得显示“账户安全”
- 有 warning 时显示“需要关注”
- 有 blocker 时首页主状态应切为“存在阻断”
- `unavailable` 必须显示
- 不显示“发起实盘行动”作为默认主按钮

------

### 页面目的

展示全账户风险，而不是只看当前 Carrier 或 BNB。

### 页面要回答

```text
当前全账户有哪些仓位？
有哪些 open orders？
有哪些 margin / balance facts？
哪些仓位有保护？
哪些仓位没有保护？
哪些事实来自 PG？
哪些事实来自交易所？
数据是否 fresh？
```

### 页面模块

1. 账户资金摘要
2. 全账户仓位表
3. 全账户挂单摘要
4. 风险状态
5. protection ownership
6. freshness 区
7. unavailable 区

### 主要接口

```
GET /api/trading-console/account-risk
```

### 展示规则

- 显示数据源
- 显示 freshness
- `unknown` risk_state 不得展示为 healthy
- `not_available` 的 PnL / margin 字段不得估算

### 页面目的

订单台账是一级页面，用于展示订单事实、订单归属、保护单组和 PG/交易所差异。

### 页面要回答

```text
哪些是本地订单？
哪些是交易所订单？
哪些订单已匹配？
哪些只在 PG？
哪些只在交易所？
哪些状态不一致？
哪些是建仓单？
哪些是保护单？
哪些是孤儿保护单？
哪些属于同一个保护单组？
哪些字段当前不可用？
```

### 页面模块

1. 分类统计
2. 订单列表
3. 保护单组视图
4. PG / Exchange 对账视图
5. 孤儿保护单视图
6. 订单详情抽屉
7. unavailable fields 区

### 主要接口

```
GET /api/trading-console/order-ledger
```

### 订单分类

- `matched`
- `pg_unchecked`
- `pg_only`
- `exchange_only`
- `mismatch`
- `orphan_protection`
- `unknown`

### 展示规则

- `pg_unchecked` 表示未请求交易所事实，不代表 clean
- `pg_only` 必须显著显示
- `exchange_only` 必须显著显示
- `mismatch` 必须显著显示
- `orphan_protection` 必须进入异常提示
- `client_order_id` 不可用时显示 `not_available`
- fee / funding / slippage 不可用时显示 `not_available`

### 页面目的

展示当前授权状态，不创建、不作废、不修改授权。

### 页面要回答

```text
是否存在授权？
授权是否可行动？
授权是否已消费？
授权是否过期？
授权是否取消？
scope 是否匹配？

未来有哪些动作槽位？
```

### 页面模块

1. 授权状态卡
2. scope 明细
3. actionable 状态
4. blocking_reason
5. future_action_slots
6. 当前不可用动作说明

### 主要接口

```
GET /api/trading-console/authorization-state
```

### 展示规则

- `is_actionable=false` 不得展示执行入口

## 9. API 使用规则

### 9.1 唯一事实源

交易控制台前端只允许调用：

```text
/api/trading-console/*
```

### 9.2 禁止调用

```text
/api/brc/*
/api/runtime/*
/api/dev/testnet/brc/*
```

除非未来主控明确把某接口纳入 trading-console contract。

### 9.3 include_exchange 规则

默认：

```text
include_exchange=false
```

含义：

```text
不读取交易所
不读取 account snapshot
返回 not_live_connected
不能展示为真实账户安全
```

当页面需要真实交易所只读事实时，可以使用：

```text
include_exchange=true
```

但前端必须展示：

```text
读取交易所事实中
读取成功 / 失败
exchange_snapshot_at
exchange_error
freshness_status
```

## 10. UI 设计原则

### 10.1 首页原则

首页以真实风险状态为中心。

不以“发起实盘行动”为中心。

### 10.2 单页主状态原则

每页只突出一个核心状态。

例如：

- 首页：当前真实风险
- 订单台账：订单一致性
- 执行控制：当前是否可执行
- 异常恢复：当前需要处理什么
- 复盘：结果与链路

### 10.3 危险动作预设原则

Gate 3 危险动作组件可以预设，但必须：

```text
disabled
unavailable
future placeholder
```

### 10.4 状态不可伪造原则

前端不得：

- 估算 not_available
- 忽略 unavailable
- 把 unknown 当 clean
- 把 warning 当 success
- 把 deferred action 当 action
- 从旧 API 拼出交易控制台事实

### 10.5 旧术语禁止原则

用户可见文案中避免：

- trial
- test authorization
- mock
- demo
- simulation
- 测试授权
- 发起测试

使用：

- 交易控制台
- 实盘行动
- 有界实盘授权
- 实盘执行
- 实盘复盘
- 异常恢复
- 订单台账
- 技术审计

## 11. AI Studio 工程范围

AI Studio 可以做：

- 页面布局
- 组件设计
- 状态卡片
- 表格
- 详情抽屉
- disabled 操作槽位
- 二次确认弹窗预设
- 只读 API 接入
- loading / empty / error / unavailable 状态
- not_available 展示
- warning / blocker 展示
- future placeholder 组件

AI Studio 不可以做：

- 接真实执行 API
- 自行寻找旧接口
- 自行调用 `/api/brc/*`
- 自行调用 `/api/runtime/*`
- 自行调用 `/api/dev/testnet/brc/*`
- 把 deferred action 做成可点击
- 根据前端状态推断可执行
- 伪造 PnL / fee / funding / slippage
- 隐藏 unknown / unavailable

## 12. 后端缺口处理规则

前端接入时，如果发现以下问题：

- 字段不足
- 状态语义不清
- 页面无法表达
- endpoint 返回结构不稳定
- not_available 影响关键页面
- unavailable 无法解释
- warning / blocker 无法映射
- BNB-first scope 影响产品表达
- 某个 future action 需要后端支持

前端窗口不直接派 Codex 后端任务。

统一输出：

```text
交易控制台后端依赖同步单 v0.2
```

交主控窗口统一吸收。

## 14. 产品验收标准

交易控制台只读版前端达到以下标准，视为产品上可接受：

1. 首页能明确展示当前是否连接真实交易所事实。
2. `not_live_connected` 不会被展示为安全。
3. `unavailable` 不会被展示为 clean。
4. 订单台账能展示 PG / exchange / mismatch / orphan protection。
5. 保护状态能展示 protected / partially / unprotected / unknown / orphaned。
6. 授权状态能展示 is_actionable / consumed / expired / cancelled / blocking_reason。
7. 执行控制页不出现可点击执行按钮。
8. Recovery 页能展示异常和 deferred action，但不执行动作。
9. 复盘页不估算 fee / funding / slippage。
10. 技术审计页能按核心 ID 查询链路。
11. Carrier Shelf 明确 BNB-first 是 v1 scope。
12. Signal marker feed 后置，不影响 P0 页面。
13. 所有页面只调用 `/api/trading-console/*`。
14. Gate 3 操作组件全部 disabled / future placeholder。
15. 页面不再使用 trial / 测试授权 作为用户可见主语言。

------

## 15. 结论

Gate 2 阶段的交易控制台产品目标是：

```text
构建一个只读、真实事实优先、风险状态清晰、订单链路可见、异常可解释、操作能力预留但不开放的 Owner 交易控制台。
```

本阶段成功标准不是“能操作”，而是：

```text
Owner 能看清楚真实风险、订单事实、授权状态、执行状态、异常状态和复盘事实；
同时不会被前端误导去执行任何未开放的真实交易动作。
```
