  max_body_ratio: 0.35      # 实体占比 ≤ 35%

**检测逻辑** (`src/domain/strategy_engine.py:184-267`):
```python
is_pinbar = (
    wick_ratio >= cfg.min_wick_ratio
    and body_ratio <= cfg.max_body_ratio

```

**问题**：只检测**几何比例**，不验证**绝对波幅**

**实际场景推测**:

ETH 15m K 线数据:
  open:  2052.50

  close: 2052.64

### 2. ATR 过滤器逻辑

**配置** (`config/core.yaml`):
```yaml
atr_volatility:
  enabled: true
  period: 14
  min_atr_ratio: 0.5      # K 线波幅 / ATR ≥ 0.5
```

**检查逻辑** (`src/domain/filter_factory.py:413-470`):
```python
def check(self, pattern: PatternResult, context: FilterContext) -> TraceEvent:
    atr = self._get_atr(kline.symbol, kline.timeframe)

    # 计算 K 线波幅与 ATR 的比率
    candle_range = kline.high - kline.low
    min_range = atr * self._min_atr_ratio

    if candle_range < min_range:
        return TraceEvent(
            passed=False,
            reason="insufficient_volatility",
            metadata={
                "candle_range": float(candle_range),
                "atr": float(atr),
                "min_required": float(min_range),
                "ratio": float(candle_range / atr),
            }
        )

    return TraceEvent(
        passed=True,
        reason="volatility_sufficient",
        metadata={
            "candle_range": float(candle_range),
            "atr": float(atr),
            "ratio": float(candle_range / atr),
        }
    )
```

**问题**：ATR 过滤器检查的是**相对比率** (`candle_range / ATR`)，而非**绝对阈值**

当整个市场波动率极低时：
- `candle_range` = 0.07 USDT (很小)
- `ATR` = 0.08 USDT (也很小)
- `ratio` = 0.07 / 0.08 = 0.875 ≥ 0.5 → **通过** ❌
