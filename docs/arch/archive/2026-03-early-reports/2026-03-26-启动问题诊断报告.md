## 问题 1: 后端循环导入错误 (严重)

### 错误信息
```
ImportError: cannot import name 'RiskConfig' from partially initialized
module 'src.application.config_manager' (most likely due to a circular import)
```

### 根本原因

`RiskConfig` 类在两个文件中重复定义：

1. **`src/domain/models.py` (line 93)** - 领域层模型
2. **`src/application/config_manager.py` (line 79)** - 应用层配置

导入链条：
```
main.py
  → config_manager.py (line 18)
    → from src.domain.models import RiskConfig
  → risk_calculator.py (line 16)
    → from src.application.config_manager import RiskConfig  # 循环！
```

### 解决方案

**方案 A (推荐)**: 统一 `RiskConfig` 定义位置

1. 删除 `src/application/config_manager.py` 中的 `RiskConfig` 类定义
2. 修改 `config_manager.py` 从 `models.py` 导入：
   ```python
   from src.domain.models import RiskConfig
   ```
3. 修改 `risk_calculator.py` 从 `models` 导入（已正确）：
   ```python
   from .models import RiskConfig
   ```

**方案 B**: 保持 `config_manager.py` 为 `RiskConfig` 唯一来源

1. 删除 `src/domain/models.py` 中的 `RiskConfig` 类定义
2. 修改 `models.py` 导入（如果需要导出）：
   ```python
   from src.application.config_manager import RiskConfig
   ```

**推荐方案 A**，因为 `RiskConfig` 是风险计算的核心配置，属于领域层概念。

### 需要修改的文件

| 文件 | 修改内容 |
|------|----------|
| `src/application/config_manager.py` | 删除 line 79-117 的 `RiskConfig` 类，添加导入 |
| `src/domain/models.py` | 保持不变（已有 `RiskConfig`） |
| `src/domain/risk_calculator.py` | 保持不变（已从 `.models` 导入） |
