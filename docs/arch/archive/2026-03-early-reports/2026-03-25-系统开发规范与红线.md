# 盯盘狗🐶 系统开发规范与红线 (STRICT_DEVELOPMENT_RULES)

> [!IMPORTANT]
> 任何参与本项目的 Agent（前端或后端）必须首先完整阅读并严格遵守本规范。

---

## 1. 后端红线 (Backend Redlines)

### 1.1 类型安全高于一切 (Pydantic SSOT)
- **禁用 `Dict[str, Any]`**：严禁在策略、过滤器等核心参数中使用泛型字典。必须定义具名的 Pydantic 类。
- **辨识联合 (Discriminator)**：多态对象（如 LogicTree 节点）必须使用 Pydantic 的 `Annotated` + `Field(discriminator='type')` 模式，确保反序列化绝对安全。
- **自动 Schema**：接口文档必须通过模型反射生成，严禁手动维护 JSON 示例。

### 1.2 实盘稳定性 (Execution Safety)
- **原子替换**：热更新配置时，必须通过 `asyncio.Lock` 保护 Runner 实例的重建与替换，绝不允许实盘数据竞争。
- **异步持久化**：主交易循环（Process Kline）中严禁出现同步的数据库 I/O 阻塞。所有记录必须通过 `asyncio.Queue` 异步落盘。

---

## 2. 前端红线 (Frontend Redlines)

### 2.1 业务逻辑“零认知” (Zero Business Logic)
- **禁止硬编码名字**：前端代码严禁出现如 `mtf`、`ema`、`pinbar` 等具体的业务指标名称或其专用表单组件。
- **基于 Schema 驱动**：所有的表单字段、步长、默认值、描述信息必须 **100% 来源于后端 `/api/strategies/meta` 接口**。
- **递归渲染**：策略构建器必须支持任意深度的逻辑树渲染（递归组件模式）。

### 2.2 UI/UX 工业设计标准
- **Apple-like 质感**：使用 `backdrop-blur`、高斯模糊、微小的圆角和细腻的阴影。
- **诊断透明度**：任何判定失败都必须展示具体的失败路径（哪棵树枝断了），不允许只显示一个简单的 `error`。

---

## 3. 协作红线 (Collaboration Rules)

### 3.1 提示词编写规范
- **明确边界**：任务提示词必须区分“领域模型修改”还是“基础设施修改”。
- **引用文档**：所有涉及重构的任务，必须在提示词末尾显式链接本 `STRICT_DEVELOPMENT_RULES.md`。

### 3.2 版本控制
- **Dev 分支先行**：所有实验性开发必须在 `dev` 分支进行。只有通过 `pytest` 冒烟测试的代码才允许合并至 `main`。

---

## 4. 质量保障与测试红线 (Testing & QA)

### 4.1 测试先行原则 (No Test, No PR)
- **后端任务**：每一个逻辑改动（策略、过滤器、API）必须在 `tests/unit` 或 `tests/e2e` 中配备对应的测试用例。
- **前端任务**：涉及核心逻辑（如 Schema 转换函数、递归渲染逻辑）的改动，必须通过 `vitest`（如有）或在 `walkthrough.md` 中提供详细的浏览器操作录屏演示。

### 4.2 强制自测要求
- **Pytest 验证**：Agent 在提交代码前，必须在本地运行相关目录的 `pytest` 并输出通过结果。
- **回归测试**：修复 Bug 的任务，必须包含一个能够复现该 Bug 的 Test Case，以防止未来发生重退化。

---

## 5. 角色职责与权限边界 (Roles & Responsibilities)

### 5.1 架构师 (Architect) - **只说不动**
- **核心职责**：负责全局方案设计、编写/修订 `implementation_plan.md`、制定物理架构及技术底座（如递归逻辑树模型、热重载流程）。
- **代码权限**：**禁止执行除文档外的任何业务代码变更**（包含但不限于 UI 展示、后端逻辑补丁、DB Schema 更新等）。
- **审查权**：对开发 Agent 提交的代码拥有“否决权”。只有架构师确认代码符合“红线”设计后，任务方可闭环。

### 5.2 开发 Agent (Developer Agent) - **只干不说**
- **核心职责**：根据架构师下发的 `SubTask` 任务书和 `implementation_plan.md` 进行具体代码实现。
- **交付义务**：必须输出代码、对应的测试用例及 `walkthrough.md` 验证文档。

---

**⚠️ 违例后果**：架构师若擅自修改业务代码，一经发现需自审并撤回；开发 Agent 若无视架构师方案擅自变更架构，代码将被全盘推倒并重新分配任务。
