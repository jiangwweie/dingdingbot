## 二、架构改进建议（6项）

### 🔴 P0 - 必须修复/实现

#### 1. 订单状态机完善（影响实盘安全）

**问题**：当前缺乏完整的订单生命周期状态机，实盘风险高。

**方案**：
```python
class OrderStateMachine:
    """订单状态机管理"""
    states = ['CREATED', 'SUBMITTED', 'PARTIAL_FILLED', 
              'FILLED', 'CANCELLED', 'REJECTED', 'EXPIRED']
```

**工时**：8h | **优先级**：P0

---

#### 2. 滑点与流动性模型（影响回测准确性）

**问题**：回测滑点固定，未考虑市场深度和价格冲击。

**方案**：
```python
@dataclass
class SlippageModel:
    """基于市场深度的滑点模型"""
    base_slippage: Decimal
    volume_impact_factor: Decimal
    volatility_factor: Decimal
```

**工时**：6h | **优先级**：P0

---

### 🟡 P1 - 重要但可延后

#### 3. 多品种组合回测与相关性分析

**问题**：当前单品种独立回测，无法验证组合策略。

**方案**：
```python
class PortfolioBacktester:
    """组合回测引擎"""
    def run_portfolio_backtest(self, 
                               strategies: List[Strategy],
                               allocations: Dict[str, Decimal]) -> PortfolioReport:
```

**工时**：16h | **优先级**：P1

---

#### 4. 策略归因分析系统
