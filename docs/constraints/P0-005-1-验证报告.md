## 四、发现的问题与修复

### 4.1 问题 1：订单 ID 混淆

**问题描述**:
- `place_order` 返回两个订单 ID：
  - `order_id`: 系统内部生成的 UUID (如 `d22dabd1-f6ce-4223-8bb5-336517454e4b`)
  - `exchange_order_id`: 交易所返回的数字 ID (如 `13018229156`)
- `cancel_order`和`fetch_order` 方法原本使用 `order_id` (UUID) 调用 CCXT API
- Binance API 只接受 `exchange_order_id` (数字 ID)

**错误信息**:
```
binance {"code":-1102,"msg":"Mandatory parameter 'orderid' was not sent, was empty/null, or malformed."}
```

**修复方案**:
1. 修改 `cancel_order`和`fetch_order` 方法的参数名为 `exchange_order_id`，明确语义
2. 更新方法文档，说明必须使用交易所订单 ID
3. 更新测试脚本，使用 `exchange_order_id` 进行查询和取消操作

**修复文件**:
- `src/infrastructure/exchange_gateway.py`
- `scripts/verify_binance_testnet.py`

## 五、关键发现

### 5.1 订单 ID 类型

| ID 类型 | 格式 | 用途 |
|---------|------|------|
| 系统订单 ID (`order_id`) | UUID (如 `d22dabd1-f6ce-4223-8bb5-336517454e4b`) | 内部追踪、前端展示 |
| 交易所订单 ID (`exchange_order_id`) | 数字 (如 `13018229156`) | API 调用（取消、查询） |

### 5.2 Binance 测试网限制

- **最小名义价值**: 100 USDT
- 测试脚本使用 0.001 BTC (约 69 USDT) 会被拒绝
- 修正为 0.002 BTC (约 138 USDT) 后成功
