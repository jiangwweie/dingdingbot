## 新功能

### 1. WebSocket 实时资产推送（S5-1）

**问题**: 之前的轮询模式延迟高（最多 60 秒）、API 请求频繁、容易触发限流。

**解决方案**: 使用 CCXT Pro WebSocket 实现实时资产推送：

- **实时推送** - 资产变化时立即接收，延迟 < 1 秒
- **降级轮询** - WebSocket 失败时自动降级到轮询模式
- **指数退避重连** - 重连延迟 1s 初始，60s 最大

**核心方法**:
```python
# 订阅 WebSocket 资产更新
await gateway.subscribe_account_updates(callback)

# 内部方法
- _ws_subscribe_account_loop()    # WebSocket 订阅循环
- _parse_ws_balance()              # 解析 WebSocket 消息
- _poll_assets_loop_with_callback() # 降级轮询
```

**配置模型**:
```python
class WebSocketAssetConfig(BaseModel):
    enabled: bool = True                    # 是否启用 WebSocket
    reconnect_delay: float = 1.0            # 初始重连延迟
    max_reconnect_delay: float = 60.0       # 最大重连延迟
    max_reconnect_attempts: int = 10        # 最大重连次数
    fallback_to_polling: bool = True        # 降级轮询
    polling_interval: int = 60              # 轮询间隔
```

### 2. 信号状态跟踪系统（S5-2）

**问题**: 之前信号生成后没有状态跟踪机制，无法查询信号从生成到成交的全流程。

**解决方案**: 实现信号全生命周期状态跟踪：

- **5 种状态** - `generated` → `pending` → `filled` / `cancelled` / `rejected`
- **状态跟踪器** - `SignalStatusTracker` 管理状态流转
- **状态查询 API** - 支持单个和批量查询

**状态流转图**:
```
信号生成 → GENERATED
    │
    ▼
等待成交 → PENDING
    │
    ├──→ 已成交 → FILLED
    ├──→ 已取消 → CANCELLED
    └──→ 被拒绝 → REJECTED
```

**API 端点**:
```python
GET /api/signals/{signal_id}/status       # 查询单个信号状态
GET /api/signals/status?status=pending    # 批量查询（可过滤）
```

**前端组件**:
```typescript
// SignalStatusBadge 组件 - 5 种状态颜色
- GENERATED: 灰色
- PENDING: 黄色
- FILLED: 绿色
- CANCELLED: 蓝色
- REJECTED: 红色
```

**相关文件**:
- `src/domain/models.py` - 新增 `SignalStatus` 枚举和 `SignalTrack` 模型
- `src/application/signal_tracker.py` - 状态跟踪器（新建）
- `src/application/signal_pipeline.py` - 集成跟踪器
- `src/infrastructure/signal_repository.py` - 添加 `signal_id` 支持
- `src/interfaces/api.py` - 状态查询 API
- `web-front/src/lib/api.ts` - 类型定义和 API 调用
- `web-front/src/components/SignalStatusBadge.tsx` - 状态徽章组件（新建）

## 技术亮点

### 1. WebSocket 降级机制

```python
async def subscribe_account_updates(self, callback):
    try:
        await self._ws_subscribe_account_loop(callback)
    except Exception as e:
        logger.warning(f"WebSocket 失败，降级到轮询：{e}")
        await self._poll_assets_loop_with_callback(callback)
```
