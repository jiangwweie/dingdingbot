1. ## 第一部分：需求说明书（完整业务版）

   ### 1. 项目基本信息

   - **项目名称**：加密货币信号监测与风险评估系统 (Crypto-Signal Monitor)
   - **核心原则**：**Zero Execution Policy**（零执行政策）。系统严禁集成任何交易下单接口，仅作为观测与通知工具。
   - **运行环境**：后端服务（Python/FastAPI），配置驱动，冷启动生效。

   ### 2. 监测范围与币种管理

   - **核心监测池（内置不可删改）**：BTC, ETH, SOL, BNB。
   - **用户自定义池（配置文件扩展）**：支持在静态配置文件中手动添加/删除其他币种（如 XRP, PEPE 等）。
   - **生效机制**：修改配置文件后，通过**重启服务**加载合并后的最终监测清单。

   ### 3. 信号识别与策略引擎 (Pinbar + EMA60)

   - **Pinbar 几何形态判定**：
     - **非颜色敏感**：判定逻辑仅基于影线、实体、全长的比例关系，不参考 K 线颜色（阴阳线均可）。
     - **核心参数**：影线占比下限、实体占比上限、实体位置偏差（均在配置文件中定义）。
   - **趋势过滤逻辑（EMA60）**：
     - **同周期过滤**：价格在 EMA60 上方仅识别看多信号，下方仅看空。
     - **开关状态**：`TREND_FILTER_ENABLED`（True/False），重启生效。

   ### 4. 多周期验证 (MTF Validation)

   - **跨周期共振**：低级别信号需校验高级别周期的 EMA60 趋势方向。
   - **开关状态**：`MTF_VALIDATION_ENABLED`（True/False），重启生效。
   - **逻辑矩阵**：支持“全开、全关、仅同周期趋势、仅 MTF 趋势”四种组合逻辑。

   ### 5. 账务查询与风险试算

   - **基础查询**：只读权限访问现货/合约余额、未实现盈亏、当前持仓及挂单。
   - **风险建模**：
     - **单笔最大损失**：基于**账户总余额的百分比**（如 1%）。
     - **建议方案**：自动生成建议开仓价、止损位，并反算建议开仓数量。
     - **杠杆熔断**：强制遵循配置文件中的 `MAX_LEVERAGE` 上限。

   ### 6. 异步通知规范

   - **推送渠道**：飞书 Webhook / 企业微信机器人。
   - **内容格式**：纯文字 Markdown。**禁止包含截图**。
   - **核心字段**：币种、周期、多空、价格、建议止损、建议仓位、MTF 状态、当前 EMA 趋势。
