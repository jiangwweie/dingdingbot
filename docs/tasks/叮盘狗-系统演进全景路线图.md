# 盯盘狗🐶 系统演进全景路线图 (ROADMAP)

本文档定义了系统从“动态监控”向“智能交易”进化的完整生命周期路径。

---

## 🟢 第一阶段：架构筑基（当前核心，SubTask E/F/C）
*   [ ] **建立“强类型”递归逻辑树 (A-1)**：后端引入 Pydantic 辨识联合，支持 3 层 `AND/OR/NOT` 嵌套，彻底封杀 `Dict` 参数漏洞。
*   [ ] **前端“全面动态化”重构**：移除硬编码组件，100% 由后端 JSON Schema 驱动表单生成。
*   [ ] **逻辑递归渲染器**：前端支持树状结构的缩进渲染，适配无限扩展的指标组合。
*   [ ] **信号标签动态化**：将 `ema_trend`、`mtf_status` 等硬编码标识重构成动态 `tags` 列表及配套数据库字段。

## 🟡 第二阶段：交互升维（工作台 2.0，SubTask B+）
*   [ ] **实现“热预览（Dry Run）”接口**：支持单笔行情即时验证逻辑树，反馈绿钩（通过）与红叉（死因）。
*   [ ] **逻辑路径可视化 (Tracing)**：在工作台直接绘制逻辑判定拓扑图，让每一个信号拦截点肉眼可见。
*   [ ] **策略模板库 CRUD**：完善 SQLite 持久化，实现策略模板的保存、查询与一键“下发到实盘”链路。

## 🟠 第三阶段：风控与执行（从监控转向交易）
*   [ ] **多周期数据对齐优化**：强化 15m/1h/4h 等多周期共振时的数据包一致性校验。
*   [ ] **动态风险头寸计算 (R)**：根据账户余额与策略风险配置，自动计算单笔推荐买入量。
*   [ ] **交易所 API 挂单集成**：对接实盘下单接口，实现“信号触发 -> 判定通过 -> 自动开仓”闭环。

## 🔵 第四阶段：工业化调优（长期稳定性）
*   [ ] **配置快照版本化**：支持策略一键回滚（Rollback）到历史稳定版本。
*   [ ] **异步 I/O 写入队列**：将高频的 Attempt 诊断日志移出主循环，通过 `asyncio.Queue` 批处理。
*   [ ] **基础指标计算缓存**：针对多策略共用指标，实施“算一次，用全局”的性能优化。
