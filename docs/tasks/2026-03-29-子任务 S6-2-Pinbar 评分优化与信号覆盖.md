## 背景与目标

### 问题背景

用户回测发现：
1. 21:00 出现看空 Pinbar（Score 0.72，十字星型，波幅 116 USDT）
2. 22:00 出现看空 Pinbar（Score 0.71，真突破型，波幅 527 USDT）

**当前问题**：
- 评分只看几何比例（影线占比），无法区分"十字星"和"真突破"
- 低质量信号（十字星）可能获得高分
- 更高质量的信号出现时，没有覆盖/替代机制

### 业务目标

1. **过滤低质量信号**：使用 ATR 门槛过滤十字星/一字线
2. **优化评分逻辑**：加入 ATR 归一化因子，反映实际波幅
3. **信号覆盖机制**：同方向更高分信号出现时，自动替代旧信号
4. **通知增强**：覆盖时明确告知用户，反向信号时提示风险

---

## 核心规则

### 1. ATR 门槛判定

```yaml
min_atr_ratio: 0.5  # K 线波幅必须 ≥ ATR × 0.5
atr_period: 14       # ATR 计算周期（业界标准）
```

**判定逻辑**：
```
candle_range = high - low
atr_ratio = candle_range / atr

if atr_ratio < 0.5:
    直接过滤，不产生信号
```

### 2. 评分公式

```python
# 基础分（几何质量）
base_score = wick_ratio  # 0~1

# ATR 加分（波幅质量）
atr_bonus = min(atr_ratio, 2.0) × 0.3

# 最终评分
final_score = base_score × 0.7 + atr_bonus
# 上限 1.0
```

**效果对比**（假设 ATR=400 USDT）：

| K 线 | wick_ratio | atr_ratio | 原始 Score | 调整后 Score |
|------|------------|-----------|------------|--------------|
| 21:00 十字星 | 0.727 | 0.29 | 0.727 | **被过滤** (0.29<0.5) |
| 22:00 真突破 | 0.715 | 1.32 | 0.715 | **0.78** |

### 3. 覆盖规则

| 条件 | 行为 |
|------|------|
| 同 symbol:timeframe:direction | 才考虑覆盖 |
| 新信号 Score > 旧信号 Score | 标记旧为 SUPERSEDED，推送新的 |
| 新信号 Score ≤ 旧信号 Score | 仍推送新的（不覆盖） |
| 反方向信号 | 不覆盖，推送时带对立提示 |

### 4. 推送频率

- **每次都推送**（除非被 ATR 过滤）
- 高分覆盖时，通知中特别说明

```python
def detect(self, kline: KlineData, atr_value: Optional[Decimal] = None) -> Optional[PatternResult]:
    """
    检测 Pinbar 形态并计算评分

    Args:
        kline: K 线数据
        atr_value: ATR 值（用于评分调整）
    """
    # ... 现有检测逻辑 ...

    # 新增：ATR 调整评分
    if atr_value and atr_value > 0:
        candle_range = kline.high - kline.low
        atr_ratio = candle_range / atr_value

        # 基础分（几何质量）
        base_score = float(wick_ratio)

        # ATR 加分（波幅质量），上限 2 倍
        atr_bonus = min(float(atr_ratio), 2.0) * 0.3

        # 最终评分
        score = base_score * 0.7 + atr_bonus
        score = min(score, 1.0)  # 上限 1.0
    else:
        score = float(wick_ratio)  # 降级为原始逻辑

    return PatternResult(
        strategy_name="pinbar",
        direction=direction,
        score=score,  # 使用调整后的评分
        details={...}
    )
```

```python
async def process_kline(self, kline: KlineData):
    # Step 1: 检测 Pinbar
    pattern_result = self._pinbar_strategy.detect(kline)
    if pattern_result is None:
        return

    # Step 2: ATR 门槛判定（新增，在 S6-2-1 实现）
    atr = self._get_atr_from_cache(kline.symbol, kline.timeframe)
    if not self._pass_atr_threshold(kline, atr):
        logger.info(f"Signal filtered by ATR: {atr_ratio:.2f} < {self._min_atr_ratio}")
        return

    # Step 3: 计算评分（ATR 调整，在 S6-2-2 实现）
    score = self._calculate_score(pattern_result, atr)

    # Step 4: 检查覆盖逻辑（新增）
    dedup_key = f"{kline.symbol}:{kline.timeframe}:{pattern_result.direction.name}"
    last_signal = await self._repo.get_active_signal(dedup_key)

    superseded_signal = None
    if last_signal and score > last_signal.score:
        # 新信号更优，标记旧信号为 SUPERSEDED
        await self._repo.update_signal_status(last_signal.signal_id, "SUPERSEDED")
        await self._repo.update_superseded_by(last_signal.signal_id, signal_id)
        superseded_signal = last_signal
        logger.info(f"Signal {last_signal.signal_id} superseded by new higher-quality signal")

    # Step 5: 保存新信号
    signal_id = await self._repo.save_signal(signal, score=score)

    # Step 6: 检查对立信号（新增）
    opposing_signal = await self._repo.get_opposing_signal(
        kline.symbol, kline.timeframe, pattern_result.direction
    )
    opposing_score = opposing_signal.score if opposing_signal else None

    # Step 7: 推送通知（在 S6-2-5 增强）
    await self._notifier.send_signal(
        signal=signal,
        superseded_signal=superseded_signal,
        opposing_score=opposing_score,
    )
```

```python
async def send_signal(
    self,
    signal: SignalResult,
    superseded_signal: Optional[SignalResult] = None,
    opposing_score: Optional[float] = None,
):
    """
    发送信号通知

    Args:
        signal: 信号结果
        superseded_signal: 被替代的旧信号（如有）
        opposing_score: 对立信号评分（如有）
    """
    # 构建消息头
    if superseded_signal:
        header = "🔥 **信号更新 - 更优品质** 🔥"
        extra_lines = [
            "",
            "━━━━━━━━━━━━━━━━━━━━━━",
            f"📊 品质评分：{signal.score:.2f} ⬆️ (前信号 {superseded_signal.score:.2f})",
            "━━━━━━━━━━━━━━━━━━━━━━",
            "",
            f"💡 说明：此信号品质高于 {superseded_signal.kline_timestamp} 的信号 ({superseded_signal.score:.2f})，已自动替代。",
        ]
    elif opposing_score:
        header = "⚠️ **反向信号提醒** ⚠️"
        extra_lines = [
            "",
            "━━━━━━━━━━━━━━━━━━━━━━",
            f"📊 品质评分：{signal.score:.2f} (对立信号 {opposing_score:.2f})",
            "━━━━━━━━━━━━━━━━━━━━━━",
            "",
            "💡 说明：与现有信号方向相反，请注意风险。",
        ]
    else:
        header = "📈 **新信号**"
        extra_lines = [
            "",
            f"📊 品质评分：{signal.score:.2f}",
        ]

    # 构建完整消息
    message = self._format_signal_message(signal, header, extra_lines)

    # 推送
    await self._push_to_feishu(message)
```

## 风险与缓解

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| ATR 计算性能 | 高频率 K 线更新可能卡顿 | 使用缓存，仅在最新 K 线闭合时计算 |
| 数据库迁移失败 | 旧数据丢失 | 迁移前备份，使用 IF NOT EXISTS |
| 并发覆盖竞争 | 两个信号同时进入 | 使用 asyncio.Lock 保护 |
| 通知轰炸 | 短时间内多条推送 | 保留冷却期逻辑，但按方向独立 |
