# 子任务E进阶开发提示词：递归组件渲染器与工作台交互重构 (SubTask E - Frontend)

**角色设定**：你是一位精通 React 递归组件设计、TailwindCSS 布局以及 JSON Schema 动态表单生成的高级前端工程师。你需要将策略工作台从“硬编码平铺模式”重构成“递归树状构建器”。

## 核心任务要求

### 1. 实现递归节点渲染器 (Recursive Builder)
- **底层组件**：创建一个 `<StrategyNode />` 递归组件。
- **逻辑门渲染**：如果节点类型是 `logic_gate`（AND/OR/NOT），渲染一个带有明显视觉边框（建议用虚线或淡色背景）的容器，并允许在内部通过 `map` 递归调用自身渲染子节点。
- **叶子节点渲染**：如果是 `trigger` 或 `filter`，根据后端下发的 Schema 动态反射出输入控件（滑块、开关、下拉等）。

### 2. 移除所有硬编码 (Zero Hardcoding)
- 彻底删除 `StrategyBuilder.tsx` 及其附属的 `PinbarParamsEditor`、`EmaFilterEditor` 等 10+ 个死板组件。
- 表单的结构、字段名、最小值、最大值、步长 (Step) 必须 **100% 来源于 `/api/strategies/meta`**。

### 3. 实现“策略热预览”交互 (Hot Preview / Dry Run)
- **功能点**：在策略编辑区显著位置增加一个“⚡️ 立即测试 (Test Now)”按钮。
- **交互流**：点击后将当前构造的逻辑树 JSON 推给 `/api/strategies/preview`。
- **反馈渲染**：将后端返回的 `Trace Graph`（结果树）通过视觉反馈展示在对应的节点上。成功的节点显示绿钩 ✅，失败的显示红叉 ❌，并悬浮展示失败明细（如 "Price 67000 < EMA 67500"）。

### 4. 默认值与快捷预设
- 当用户新增一个节点时，优先读取 Schema 中的 `default` 字段填充初始状态，确保用户“点开即用”。

## 视觉标准 (UI/UX)
- 保持 Apple-like 的极简质感：使用 `backdrop-blur`、圆角矩形和细腻的阴影。
- 逻辑嵌套的缩进要清晰，方便用户一眼看出“谁是谁的子条件”。
- 保证在嵌套 3 层时界面不发生布局崩坏。

## 行动指南
你不需要碰后端代码，你的任务是**“让前端变成一个通用的、能够解析任何递归逻辑树并实施可视化诊断的超级终端”**。完成后，请确保能在界面上成功拼装出一个包含 (AND (OR Trigger1 Trigger2) Filter1) 的复杂策略。
