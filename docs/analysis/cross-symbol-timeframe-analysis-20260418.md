## 三、关键问题诊断

### 3.1 问题 1: 所有交易都是逆势（100% 逆势）⚠️ 严重

#### 现象

```
alignment_stats:
  - aligned_trades: 0
  - against_trend_trades: 298
  - 逆势交易占比: 100.0%
```

#### 根因分析

**1. EMA 趋势判断逻辑错误**:

示例数据（BNB 15m）:
```
EMA component:
  - price: 624.54
  - EMA: 621.47
  - distance: 0.49% above EMA
  - score: 0.099 (很低)
```

问题:
- 价格在 EMA 附近（0.18%-2.19%），距离太近
- score 很低（0.036-0.202），说明趋势不明确
- 可能误判为逆势交易

**2. 归因计算逻辑问题**:
- `alignment_stats` 计算基于 EMA 方向 vs 信号方向
- 但 EMA 方向判断可能不准确（价格在 EMA 附近震荡）

#### 影响

- 逆势交易胜率低（理论上应 < 40%）
- 实际胜率 40.2%，符合逆势交易特征
- 导致整体亏损

#### 建议

**P0 优先级（立即修复）**:

1. **检查 EMA 趋势判断逻辑**:
   - 文件: `src/domain/filter_factory.py` - `EmaTrendFilterDynamic`
   - 确认趋势判断逻辑是否正确

2. **修复 alignment_stats 计算逻辑**:
   - 文件: `src/application/attribution_analyzer.py`
   - 确认顺势/逆势判断标准

3. **或者禁止逆势交易**:
   - 在过滤器中强制要求顺势交易
   - 逆势信号直接拒绝

### 3.2 问题 2: 高分信号胜率低（28.4%）⚠️ 严重

#### 现象

| 报告 | 高分信号数 | 胜率 | 平均盈亏比 | 报告总收益 |
|------|-----------|------|-----------|-----------|
| BTC 15m | 65 | 36.4% | -0.27 | -47.11% |
| ETH 15m | 53 | 31.8% | -0.36 | -48.91% |
| SOL 15m | 47 | 47.5% | -0.05 | +1.53% |
| BTC 1h | 18 | 41.2% | -0.18 | -4.04% |
| ETH 1h | 14 | 50.0% | 0.00 | -2.26% |
| SOL 1h | 19 | 47.4% | -0.05 | +1.03% |

**关键发现**:
- 高分信号（>0.7）共 252 个，平均胜率仅 **28.4%**
- 归因分数与实际盈亏完全脱钩

#### 根因分析

**1. 归因分数计算问题**:

```
final_score = pattern × 0.55 + ema × 0.25 + mtf × 0.20
```

问题:
- pattern score 权重 55%，但形态质量高 ≠ 交易盈利
- EMA/MTF 权重 45%，但都在逆势交易
- 结果: **高分形态 + 逆势交易 = 低胜率**

**2. 归因维度独立，未考虑组合效应**:

示例:
```
信号 A:
  - pattern: 0.85 (高分)
  - ema_trend: 0.05 (逆势)
  - mtf: 1.0 (对齐)
  → final_score: 0.78 (中高分)
  → 实际: 逆势交易，胜率低
```

问题:
- 归因各维度独立评分
- 没有考虑: 高分形态 + 逆势交易 = 糟糕组合

#### 建议

**P1 优先级（1 周内）**:

1. **归因分数应包含趋势方向因子**:
   ```python
   if 逆势:
       final_score × 0.5  # 大幅降分
   if 顺势:
       final_score × 1.2  # 适当加分
   ```

2. **增加「归因分数 → 实际盈亏」映射表验证**:
   - 每次回测后生成分数区间统计
   - 验证归因分数是否真的能预测盈亏

3. **多维度交叉分析**:
   - 形态质量 × EMA 强度 → 胜率热力图
   - 发现最优组合区间

### 3.3 问题 3: 盈亏比分布异常 ⚠️ 警告

#### 现象

| 盈亏比区间 | 次数 | 胜率 | 说明 |
|-----------|------|------|------|
| high_rr (>2:1) | **0** | 0% | 没有高盈亏比交易 |
| medium_rr (1:1-2:1) | 95 | **100%** | 全部盈利 |
| low_rr (<1:1 盈利) | **0** | 0% | 没有低盈亏比盈利交易 |
| stop_loss (<0:1) | 151 | **0%** | 全部亏损 |

**关键发现**:
- 实际胜率: 95/(95+151) = **38.6%**
- 没有高盈亏比交易（TP2/TP3 从未触发）
- TP1 后剩余仓位全部止损

#### 根因分析

**1. TP1 平仓后剩余仓位全部止损**:

```
TP1 触发: 95 次 → 全部盈利
剩余仓位: 最终全部触发 SL → 亏损
```

问题:
- TP1 平仓比例过低（可能只有 30-40%）
- 剩余仓位风险未控制
- 导致盈利交易变成亏损

**2. 没有高盈亏比交易**:

问题:
- 理论上应该有 TP2/TP3 触发的高盈亏比交易
- 但实际没有，说明 TP2/TP3 从未触发
- 可能原因:
  - TP2/TP3 配置问题
  - 市场条件不满足
  - 止损先于 TP2/TP3 触发

#### 建议

**P0 优先级（立即修复）**:

1. **提高 TP1 平仓比例至 60-70%**:
   - 减少 TP1 后剩余仓位风险
   - 保护盈利

2. **检查 TP2/TP3 配置和触发逻辑**:
   - 文件: `src/domain/matching_engine.py`
   - 确认 TP2/TP3 是否正确配置
   - 检查触发条件是否合理

3. **考虑追踪止损（Trailing Stop）**:
   - TP1 后启动追踪止损
   - 保护剩余仓位盈利

## 五、归因系统改进建议

### 5.1 当前归因系统的核心问题

| 问题 | 说明 | 影响 |
|------|------|------|
| 归因是「描述性统计」 | 只能事后解释，不能事前预测 | 无法指导交易决策 |
| 归因维度与交易结果脱钩 | 高分信号胜率低 | 归因分数无意义 |
| 归因维度独立 | 未考虑组合效应 | 无法发现最优组合 |

### 5.2 改进方案

#### 方案 1: 增加「归因分数 → 实际盈亏」映射表

**实现方式**:

```python
# 每次回测后生成
score_performance = {
    "[0.9, 1.0]": {
        "count": 5,
        "win_rate": 0.60,
        "avg_pnl": 15.3,
        "suggestion": "加仓"
    },
    "[0.7, 0.9)": {
        "count": 10,
        "win_rate": 0.40,
        "avg_pnl": 3.2,
        "suggestion": "标准仓位"
    },
    "[0.0, 0.7)": {
        "count": 15,
        "win_rate": 0.20,
        "avg_pnl": -8.5,
        "suggestion": "跳过"
    }
}
```

**效果**:
- 验证归因分数是否真的能预测盈亏
- 提供可操作的决策依据

#### 方案 2: 增加趋势方向因子

**实现方式**:

```python
# 当前计算
final_score = pattern × 0.55 + ema × 0.25 + mtf × 0.20

# 改进后
base_score = pattern × 0.55 + ema × 0.25 + mtf × 0.20

if 逆势:
    final_score = base_score × 0.5  # 大幅降分
elif 顺势:
    final_score = base_score × 1.2  # 适当加分
else:
    final_score = base_score  # 震荡，不调整
```

**效果**:
- 逆势交易分数降低，自动过滤
- 顺势交易分数提高，优先交易

#### 方案 3: 归因驱动的动态风控

**实现方式**:

```python
# 仓位调整公式
position_multiplier = (
    base_position
    × score_factor      # 0.5-1.5 根据分数
    × trend_factor      # 0.5-1.2 根据顺势/逆势
)

# 示例
if final_score > 0.9 and 顺势:
    position_multiplier = 1.5  # 加仓
elif final_score < 0.7 or 逆势:
    position_multiplier = 0.0  # 跳过
else:
    position_multiplier = 1.0  # 标准仓位
```

**效果**:
- 高质量信号加仓，提高盈利
- 低质量信号跳过，避免亏损
