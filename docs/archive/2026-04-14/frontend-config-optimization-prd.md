> **文档版本**: 1.0

> **产品负责人**: 用户（独立交易员）  
> **开发优先级**: P0

---

## 一、产品概述

### 1.1 产品背景

回测配置 KV 化开发已完成（132 测试通过，代码审查 97/100），现在需要优化前端配置页面，解决以下问题：

1. **配置分散** - 用户需要访问 3 个不同页面才能完成完整配置
2. **分类不清** - 全局配置、策略配置、回测配置混在一起
3. **信息架构复杂** - ConfigProfiles 一个页面有 4 个 Tab，内容深度不一致

### 1.2 目标用户

| 用户类型 | 特点 | 需求 |
|---------|------|------|
| **独立交易员** (单人使用) | 自己配置、自己交易、自己回测 | 快速配置、清晰分类、易于切换 |

### 1.3 核心原则

1. **配置分层** - 按影响范围和变更频率分类
2. **快速访问** - 常用配置放在容易访问的位置
3. **避免冗余** - 回测配置默认不保存，避免配置爆炸

## 二、配置层级定义

### Level 1: 全局系统配置（重启生效）

**特点**: 影响整个系统运行，设置后很少更改

| 配置项 | 默认值 | 调整频率 |
|--------|--------|----------|
| `queue_batch_size` | 10 | 极低 |
| `queue_flush_interval` | 5.0 秒 | 极低 |
| `queue_max_size` | 1000 | 极低 |
| `warmup_history_bars` | 100 | 极低 |
| `signal_cooldown_seconds` | 14400 (4h) | 极低 |

**前端策略**: 折叠/隐藏在高级设置中，减少干扰

---

### Level 2: 策略级配置（热重载生效）

**特点**: 影响策略信号生成，需要根据市场条件调整

| 配置类别 | 配置项 | 默认值 | 调整频率 |
|---------|--------|--------|----------|
| **形态识别** | `pinbar.min_wick_ratio` | 0.6 | 周/月 |
| | `pinbar.max_body_ratio` | 0.3 | 周/月 |
| **过滤器** | `ema.period` | 60 | 周/月 |
| | `mtf_ema_period` | 60 | 周/月 |
| | `atr_period` | 14 | 周/月 |
| **风控** | `max_loss_percent` | 0.01 (1%) | 月/季 |
| | `max_leverage` | 10 | 月/季 |

**前端策略**: 主入口，放在策略配置页面，易于访问

---

### Level 3: 回测临时配置（单次生效）

**特点**: 仅影响当次回测结果，用于测试不同参数组合

| 配置项 | 默认值 | 用途 |
|--------|--------|------|
| `symbol` | BTC/USDT:USDT | 测试不同币种 |
| `timeframe` | 15m | 测试不同周期 |
| `startTime/endTime` | 最近 30 天 | 测试不同时间段 |
| `slippage_rate` | 0.001 (0.1%) | 测试滑点影响 |
| `fee_rate` | 0.0004 (0.04%) | 测试手续费影响 |
| `initial_balance` | 10000 USDT | 测试不同资金规模 |

**前端策略**: 回测页面快速切换，默认不保存

## 三、需求详情

### 3.1 导航结构优化

**当前结构**:
```
主导航
├── 监控中心
├── 策略回测 (下拉菜单)
├── 交易管理 (下拉菜单)
└── 系统设置 (下拉菜单)
    └── 配置快照 → ConfigProfiles (4 个 Tab)
```

**新结构**:
```
主导航
├── 📊 监控中心
├── 🧪 回测沙箱 (独立页面)
├── ⚙️ 策略配置 (独立页面)
└── 🔧 系统设置
    └── Profile 管理
```

**改动**:
1. 策略配置从 ConfigProfiles 中独立出来，成为一级导航
2. 回测沙箱保持独立页面
3. 系统设置仅保留 Profile 管理、备份恢复

---

### 3.2 策略配置页面（新建 `/strategies`）

**页面结构**:
```
策略配置 (/strategies)
├── 策略列表
│   ├── 策略名称
│   ├── 启用/禁用切换
│   ├── 快速编辑按钮
│   └── [新建策略] 按钮
│
├── 策略编辑表单（点击策略后展开）
│   ├── 基础信息
│   │   ├── 策略名称
│   │   ├── 描述
│   │   └── 作用域 (币种 + 周期)
│   │
│   ├── 触发器参数 (Level 2)
│   │   ├── Pinbar 参数
│   │   ├── Engulfing 参数
│   │   └── ...
│   │
│   ├── 过滤器链 (Level 2)
│   │   ├── EMA 趋势
│   │   ├── MTF 确认
│   │   ├── ATR 过滤
│   │   └── ...
│   │
│   └── 风控参数 (Level 2)
│       ├── max_loss_percent
│       └── max_leverage
│
└── 策略模板
    ├── 保存为模板
    └── 从模板加载
```

**功能需求**:
| 功能 | 说明 | 优先级 |
|------|------|--------|
| 策略列表 | 显示所有策略，支持启用/禁用切换 | P0 |
| 策略编辑 | 编辑策略参数，实时保存 | P0 |
| 策略模板 | 保存/加载策略参数预设 | P1 |
| 配置验证 | 参数范围校验、冲突检测 | P1 |

### 3.3 回测沙箱页面（优化现有 `/backtest`）

**页面结构**:
```
回测沙箱 (/backtest)
├── 快速配置（不保存）
│   ├── 币种：[下拉选择，7 个选项]
│   ├── 周期：[下拉选择，7 个选项]
│   └── 时间范围：[快速日期选择器]
│
├── 高级配置（折叠）
│   ├── 策略组合
│   ├── 风控覆写
│   └── 滑点/手续费配置
│
├── [保存为预设] 按钮（可选）
│
└── 回测结果
    ├── 回测报告
    └── 使用的配置参数快照（自动记录）
```

**功能需求**:
| 功能 | 说明 | 优先级 |
|------|------|--------|
| 快速配置 | 币种、周期、时间范围快速选择 | P0 |
| 高级配置 | 折叠，点击展开 | P1 |
| 保存预设 | 用户主动点击才保存 | P2 |
| 参数快照 | 回测报告自动记录使用的参数 | P0 |

**不做的功能**:
- ❌ 回测配置自动保存（避免配置爆炸）
- ❌ 回测配置与策略配置混用

### 3.5 配置项 Tooltip 完善

**需求**: 为所有 Level 2 配置项添加完整的 tooltip 和默认值说明

**示例**:
```
Pinbar 最小影线比 [ℹ️]
────────────────────────────
默认值：0.6 (60%)
范围：0.5 - 0.7

说明：影线长度占整个 K 线范围的比例。
较高的值会选择更明显的 Pinbar 形态，
但可能会错过一些机会。

调整建议：
- 高波动市场：降低到 0.5
- 低波动市场：提高到 0.7
────────────────────────────
```

**覆盖范围**:
- 所有形态识别参数（Pinbar/Engulfing/Doji/Hammer）
- 所有过滤器参数（EMA/MTF/ATR/成交量）
- 所有风控参数

## 七、技术注意事项

### 7.1 配置同步

- 策略配置修改后，需要调用 `PUT /api/strategy/params` 保存
- 系统配置修改后，需要调用 `PUT /api/config/*` 保存
- 配置变更后，需要触发热重载通知

### 7.2 回测参数快照

- 回测 API 返回的报告中已包含 `used_configs` 字段
- 前端需要在回测结果页面展示该字段

### 7.3 现有代码复用

- `StrategyParamPanel.tsx` - 可复用到策略配置页面
- `SystemTab.tsx` - 可迁移到系统设置页面
- `QuickDateRangePicker.tsx` - 可复用到回测页面
