# 前端开发任务提示词：补齐策略作用域配置与一键下发链路

**角色设定**：你是一位精通 React/TypeScript 以及前端状态管理的资深前端开发工程师。目前由你接手“策略组装工作台”模块的紧急功能补齐任务。

## 任务背景与痛点
我们在对后端实盘监控引擎的联调中发现，虽然前端成功实现了“策略模板”的搭建和保存，但存在两个阻断实盘运行的严重问题：
1. **缺失环境隔离配置**：前端表单页缺乏让用户指定“该策略对哪些币种、哪些时间周期生效”的作用域表单域。这会导致策略默认全局生效（`is_global: true`），引发灾难性的跨品种误报。
2. **缺乏一键应用链路（未生效）**：前端调用 `POST /api/strategies` 仅仅是把模板存到了后端的 SQLite 模板库里，底层跑的实盘引擎（配置在 `user.yaml` / `/api/config`）依然是旧的。必须要把策略追加到“运行池”中才会真正生效。

## 核心开发任务书

### 任务一：补齐作用域配置表单（环境隔离）
**目标文件**：`web-front/src/components/StrategyBuilder.tsx` (或者相关的策略参数编辑组件)
**需求描述**：
为策略对象增加对 `is_global` (boolean) 和 `apply_to` (string 数组) 两个字段的收集能力。

**UI/UX 建议**：
1. 新增一个 Switch 组件（例如：“应用于全局所有币种和周期”），绑定到 `is_global` 字段，默认开启 (`true`)。
2. 当 Switch 被关闭 (`is_global` === `false`) 时，下方弹出一个穿梭框、下拉多选框或是动态添加列表（视你组件库方便而定），让用户指定目标。
3. 值的格式必须严格按照 `符号/计价币:本位币:时间周期` 进行拼接。
   - 示例：`BTC/USDT:USDT:15m` 
   - 示例代码逻辑：可以将系统获取到的 `symbols` 和 `timeframes` 提供给用户选择，选中后用冒号拼接进 `apply_to` 数组。

### 任务二：打通“下发到实盘”的数据链路
**目标文件**：策略列表页组件或工作台组件
**需求描述**：
增加“一键应用到实盘”按钮（可以在列表页每项操作列，或在保存策的时候以弹窗询问）。

**核心交互逻辑（必须严格遵守的 3 步走）**：
1. 先通过请求 `GET /api/config` 拿到当前实盘真实在跑的全量配置。
2. 从刚才的配置中拿出 `config.active_strategies` 数组，将用户选中的这套“自定义策略”（其数据结构 `StrategyDefinition`）**append 到数组末尾**（如果不想重名，可以通过 id 把旧的同名策略滤掉）。
3. 组装 Payload 为 `{ "active_strategies": [合并后的新数组] }` 格式，发起 `PUT /api/config` 请求。
4. 后端返回 200 成功后，给出全局 Toast 提示：“策略已挂载，实盘引擎热重载成功！”

**注意事项**：你此次修改不需要了解后端通信底层细节，只需要按照上述接口逻辑处理好前端表单与请求下发。请确保 TypeScript 类型定义已更新这几个字段。
