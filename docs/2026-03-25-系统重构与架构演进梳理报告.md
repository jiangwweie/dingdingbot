# 系统重构与架构演进梳理报告 (2026-03-25)

## 1. 系统使命 (System Mission)
构建一个**完全动态化、高并发、强状态一致性**的加密货币量化交易信号监控与回测沙箱系统。
打破传统硬编码的限制，允许高级交易员通过“积木化”的零代码前端工作台，自由拼装多态的触发器（Triggers）与拦截滤网（Filters），形成环境隔离的组合策略。不仅能通过历史数据极速回测验证胜率，更支持**“一键下发、无缝挂载、实时实盘热重载”**，并在毫秒级的推流中保障状态防断层与并发安全。

---

## 2. 前后端开发规范与架构规范 (Architecture Standards)

### 后端架构规范 (Backend)
- **单一事实来源 (SSOT)**：内存中运行的 `user_config` 永远是唯一生效配置，所有的实盘运算必须依赖于该指针，杜绝各模块自行读取硬盘配置。
- **快照隔离与原子热重载**：在响应配置变更时，基于观察者模式 (Observer Pattern) 新建独立的 `DynamicStrategyRunner` 实例并填充历史状态，准备就绪后使用 `asyncio.Lock()` 一次性完成指针替换，消灭实盘数据竞争。
- **异步非阻塞 IO (Fire-And-Forget)**：任何高频写入数据（如 Attempt 测算诊断记录），禁止同步堵塞主 WebSocket 事件循环，必须压入 `asyncio.Queue` 由后台 Worker 异步批处理落盘。
- **动态强校验 (Pydantic 辨识联合)**：领域模型坚决防范裸漏的 `Dict[str, Any]`。所有的动态入参必须使用类型安全的多态实体类继承（如 `PinbarParams` / `EmaFilterParams`）进行反序列化约束。

### 前端开发规范 (Frontend)
- **Schema-Driven UI (架构驱动渲染)**：坚决杜绝在视图层硬编码任意一个带有具体业务名字的（如 `mtf`, `ema`）渲染组件！所有动态表单必须且只基于后端下发的 `/api/strategies/meta` (JSON Schema) 进行全自动反射遍历渲染。
- **单向数据流与状态控制**：组件渲染基于扁平化的 JSON 定义进行联动。
- **面向扩展编程**：若后端扩展新的“RSI 过滤器”，前端应做到**代码零修改、静默支持、原生拼装**。

## 3. 系统核心功能 (Core Features)

### ✅ 已完成功能 (Completed)
1. **回测沙箱体系**：历史 K 线数据摄取、完整生命周期的模拟撮合运算、多时间周期融合诊断。
2. **多模触发策略引擎**：单策略持有多 Trigger（形态）组合，并且下挂任意长 Filter（筛网）调用链判定。
3. **环境隔离与真空降级**：新配置的策略可通过 `apply_to` 机制划定专属运行势力圈，并在没有配置专属标的时实施静默真空跳过，防止误报。
4. **数据库与接口底座设**：基于 SQLite 的信号大盘展示接口、`custom_strategies` 库模板策略持久化 CRUD。
5. **消息投递中枢**：Feishu、Telegram 第三方实时推送告警。
6. **热更新机制实现**：底层完成引擎热插拔与状态回填闭环。

### ⏳ 未完成功能 (Pending / W.I.P)
1. 前端：依据后端 Schema 重写为 100% 动态拼装的策略工作台 (Dynamic Strategy Builder)。
2. 前端：打通策略工作台中的“保存模板 -> 一键应用到实盘”合并下发链路。
3. 后端：剥离硬编码 `ema_trend` 标签，为信号发送卡片装配动态 Json Tags。
4. 后端：对 Pydantic 的字典参数进行防线收紧升级（替换为带有模型推导生成的验证体）。

## 4. 未来拓展功能 (Future Expandability)
按照现有地基，未来将平滑支持以下进阶演进：
1. **抽象 AST 表达式树**：支持嵌套定义高度复杂的组合，如 `(Pinbar AND EMA) OR (吞没 AND MACD)`。
2. **策略入库一键干跑评价 (One-Click Dry Run)**：一键应用前先在沙盒快速验证这套策略过去 30 天胜率，不亏钱再装向实盘。
3. **策略单体快照与回滚 (Versioning & Rollback)**：保存自定义模板时采用追加快照 (Append-Only) ，支持实盘崩盘后的一键闪回系统。
4. **资产/账户组级别的风险隔离**：对每个不同监控策略下的执行信号，独立核算分配不同级别的风险限额。
