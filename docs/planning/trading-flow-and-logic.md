## 2. 详细判断逻辑（数值示例）

### 示例场景

假设当前市场状态：

```
ETH/USDT:USDT 1h K 线：
- timestamp: 2024-01-15 10:00:00
- open: 2500 USDT
- high: 2520 USDT
- low: 2480 USDT
- close: 2515 USDT
- is_closed: True

账户状态：
- available_balance: 10000 USDT
- last_signal_time: 2024-01-15 05:30:00

EMA 状态：
- EMA(50) on 1h: 2490 USDT
- EMA(60) on 4h: 2470 USDT

上一根已闭合 4h K 线：
- close: 2485 USDT
```

---

### Step 1: Pinbar 形态检测

**计算**：

```
K 线范围 = high - low = 2520 - 2480 = 40 USDT

下影线 = low - min(open, close) = 2480 - min(2500, 2515) = 2480 - 2500 = -20 USDT
实际下影线 = min(open, close) - low = 2500 - 2480 = 20 USDT

实体 = |close - open| = |2515 - 2500| = 15 USDT

下影线占比 = 20 / 40 = 0.5  ❌ 不满足 ≥ 0.6
实体占比 = 15 / 40 = 0.375  ❌ 不满足 ≤ 0.3
```

**判断**：未检测到 Pinbar → **结束**

---

### 假设检测到 Pinbar（调整示例数据）

调整后的 K 线：

```
open: 2505 USDT
high: 2520 USDT
low: 2480 USDT
close: 2510 USDT
```

**计算**：

```
K 线范围 = 2520 - 2480 = 40 USDT

下影线 = min(open, close) - low = 2505 - 2480 = 25 USDT
实体 = |close - open| = |2510 - 2505| = 5 USDT

下影线占比 = 25 / 40 = 0.625  ✅ ≥ 0.6
实体占比 = 5 / 40 = 0.125  ✅ ≤ 0.3

实体位置：
实体上边界 = max(open, close) = 2510 USDT
实体下边界 = min(open, close) = 2505 USDT
K 线中点 = (high + low) / 2 = (2520 + 2480) / 2 = 2500 USDT

实体在中点上方 → 看涨 Pinbar ✅
```

**结果**：检测到看涨 Pinbar → **进入 Step 2**

---

### Step 2: EMA 趋势过滤

**计算**：

```
当前价格 = close = 2510 USDT
EMA(50) = 2490 USDT

价格在 EMA 上方：
2510 > 2490  ✅

EMA 距离：
distance = |2510 - 2490| / 2490 = 20 / 2490 = 0.00803 = 0.803%
distance ≥ 0.005 (0.5%)  ✅
```

**判断**：价格在 EMA 上方且距离足够 → **进入 Step 3**

---

### Step 3: MTF 趋势确认

**计算**：

```
当前周期 = 1h
高周期 = 4h（MTF 映射：1h → 4h）

上一根已闭合 4h K 线：
- 4h_close = 2485 USDT
- 4h_EMA(60) = 2470 USDT

4h 趋势判断：
2485 > 2470  → 4h BULLISH ✅

方向匹配：
LONG 信号 + 4h BULLISH  → 通过 ✅
```

**逻辑解释**：
- **顺大逆小**：小周期（1h）做反转（Pinbar 看涨），大周期（4h）要顺势（上涨趋势）
- 1h Pinbar 在下跌末端反转（逆小）+ 4h 上涨趋势（顺大）→ 符合逻辑

**结果**：MTF 趋势确认通过 → **进入 Step 4**

---

### Step 4: 冷却去重检查

**计算**：

```
上一次信号时间 = 2024-01-15 05:30:00
当前时间 = 2024-01-15 10:00:00

时间差 = 10:00:00 - 05:30:00 = 4.5 小时 = 16200 秒
冷却时间 = 14400 秒（4 小时）

16200 ≥ 14400  ✅
```

**结果**：冷却期已过 → **进入 Step 5**

---

### Step 5: 生成交易信号

**计算**：

```
入场价 = close = 2510 USDT
止损价 = low = 2480 USDT（Pinbar 最低点）
风险距离 = |2510 - 2480| = 30 USDT
```

**输出信号**：

```json
{
  "symbol": "ETH/USDT:USDT",
  "timeframe": "1h",
  "direction": "LONG",
  "entry_price": 2510.00,
  "stop_loss": 2480.00,
  "strategy": "Pinbar + EMA(50) + MTF(4h)",
  "timestamp": "2024-01-15T10:00:00Z"
}
```

**结果**：信号生成 → **进入 Step 6**

---

### Step 6: 仓位计算

**计算参数**：

```
账户余额 = 10000 USDT
单笔最大损失 = 1% (max_loss_percent = 0.01)
最大杠杆 = 20 (max_leverage = 20)
```

**计算公式**：

```
Step 1: 计算风险金额
risk_amount = balance × max_loss_percent
risk_amount = 10000 × 0.01 = 100 USDT

Step 2: 计算止损距离
stop_distance = |entry_price - stop_loss|
stop_distance = |2510 - 2480| = 30 USDT

Step 3: 计算仓位大小
position_size = risk_amount / stop_distance
position_size = 100 / 30 = 3.33 ETH

Step 4: 计算仓位价值
position_value = position_size × entry_price
position_value = 3.33 × 2510 = 8358.3 USDT

Step 5: 计算杠杆
leverage = position_value / balance
leverage = 8358.3 / 10000 = 0.84 倍

Step 6: 杠杆约束检查
leverage ≤ max_leverage (20)  ✅ 通过
```

**最终仓位**：

```
position_size = 3.33 ETH
position_value = 8358.3 USDT
leverage = 0.84 倍
risk_amount = 100 USDT (1% 账户余额)
```

**重要说明**：

1. **仓位由风险金额决定**，不是由仓位比例限制决定
2. **系统没有 `max_position_percent` 限制**，只有两个约束：
   - 风险约束：单笔最大损失 = 1% 账户余额
   - 杠杆约束：最大杠杆 = 20 倍
3. **止损距离越小，仓位越大**（但风险金额固定为 100 USDT）
4. **实际杠杆 0.84 倍远低于限制**，说明仓位合理

**结果**：仓位计算完成 → **进入 Step 7**

---

### Step 7: 订单执行

**计算止盈目标**：

```
入场价 = 2510 USDT
止损价 = 2480 USDT
风险距离 = 30 USDT

TP1 目标价 = 2510 + 1.0 × 30 = 2540 USDT
TP2 目标价 = 2510 + 3.5 × 30 = 2615 USDT

TP1 平仓量 = 0.7968 × 50% = 0.3984 ETH
TP2 平仓量 = 0.7968 × 50% = 0.3984 ETH
```

**订单列表**：

| 订单类型 | 方向 | 价格 | 数量 | 说明 |
|---------|------|------|------|------|
| ENTRY | LONG | 2510 USDT | 3.33 ETH | 开仓 |
| TP1 | SELL | 2540 USDT | 1.665 ETH | 第一止盈（+1.0R） |
| TP2 | SELL | 2615 USDT | 1.665 ETH | 第二止盈（+3.5R） |
| SL | SELL | 2480 USDT | 3.33 ETH | 止损（全部剩余仓位） |

**注意**：
- `breakeven_enabled = False` → TP1 成交后，SL 保持 2480 USDT（不移至 2510 USDT）

**结果**：订单已提交 → **进入 Step 8**

---

### Step 8: 订单生命周期

**场景 1：价格触及 TP1（2540 USDT）**

```
触发条件：价格 ≥ 2540 USDT

执行：
1. TP1 成交，平仓 1.665 ETH
2. 盈亏计算：
   pnl = (2540 - 2510) × 1.665 = 30 × 1.665 = 49.95 USDT
   fee = 2540 × 1.665 × 0.0405% = 1.71 USDT
   net_pnl = 49.95 - 1.71 = 48.24 USDT

3. 剩余仓位：1.665 ETH
4. SL 保持原位：2480 USDT（不移动）
5. TP2 继续挂单：2615 USDT

状态：持仓中，等待 TP2 或 SL
```

---

**场景 2：价格继续上涨触及 TP2（2615 USDT）**

```
触发条件：价格 ≥ 2615 USDT

执行：
1. TP2 成交，平仓剩余 1.665 ETH
2. 盈亏计算：
   pnl = (2615 - 2510) × 1.665 = 105 × 1.665 = 174.83 USDT
   fee = 2615 × 1.665 × 0.0405% = 1.76 USDT
   net_pnl = 174.83 - 1.76 = 173.07 USDT

3. 总盈亏：
   total_pnl = 48.24 + 173.07 = 221.31 USDT
   total_risk = 100 USDT
   R_multiple = 221.31 / 100 = 2.21R

状态：交易结束，全部平仓
```

---

**场景 3：价格触及 SL（2480 USDT）**

```
触发条件：价格 ≤ 2480 USDT

执行：
1. SL 成交，平仓全部剩余仓位
2. 盈亏计算：

   情况 A：TP1 未成交
   pnl = (2480 - 2510) × 3.33 = -30 × 3.33 = -99.9 USDT
   fee = 2480 × 3.33 × 0.0405% = 1.34 USDT
   net_pnl = -99.9 - 1.34 = -101.24 USDT
   R_multiple = -101.24 / 100 = -1.01R

   情况 B：TP1 已成交，剩余仓位被止损
   pnl = (2480 - 2510) × 1.665 = -30 × 1.665 = -49.95 USDT
   fee = 2480 × 1.665 × 0.0405% = 0.67 USDT
   net_pnl = -49.95 - 0.67 = -50.62 USDT
   total_pnl = 48.24 - 50.62 = -2.38 USDT
   R_multiple = -2.38 / 100 = -0.02R

状态：交易结束，全部平仓
```

## 3. 关键参数说明

### 3.1 为什么 EMA 周期是 50？

**搜索结果**：
- EMA=60: 2024 +5168, 2025 +1645
- **EMA=50**: 2024 +5952, 2025 +4399 ✅ 更优
- EMA=40: 信号过多，质量下降

**解释**：
- EMA 周期越小，对趋势变化响应越快
- 50 是在"响应速度"和"噪音过滤"之间的最优平衡

---

### 3.2 为什么移除 ATR 过滤器？

**验证结果**：
- 有 ATR (max_atr_ratio=0.01): 217 笔交易
- 无 ATR: 217 笔交易（完全相同）

**结论**：
- ATR 过滤器在当前策略下完全冗余
- 移除后简化逻辑，不影响性能

---

### 3.3 为什么 TP 目标是 [1.0, 3.5]？

**对比测试**：
- TP=[1.0, 2.5]: 2024 +5168, 2025 +1645
- **TP=[1.0, 3.5]**: 2024 +5952, 2025 +4399 ✅ 显著更优
- TP=[1.0, 1.5]: 恶化 41% ❌

**解释**：
- TP2=3.5R 捕获大行情，是主要盈利来源
- 16% 的交易命中 TP2，贡献了大部分利润

---

### 3.4 为什么不启用 Breakeven？

**对比测试**：
- BE=ON: 3 年 -15723 USDT
- **BE=OFF**: 3 年 -10116 USDT ✅ 改善 +5607 (36%)

**解释**：

- 保持 SL 在原位，让交易有更多"呼吸空间"

**结论**：
- 策略在 ETH 1h 上表现最稳定
- 其他币种/周期不适用

## 4. 风险控制要点

### 4.1 单笔风险

```
最大损失 = 1% 账户余额
示例：10000 USDT → 最大损失 100 USDT
```

### 4.2 杠杆限制

```
最大杠杆 = 10 倍
实际杠杆 ≈ 0.8 倍（远低于限制）
```

### 4.3 冷却时间

```
冷却时间 = 4 小时
目的：避免同一趋势中重复开仓
```

### 4.4 止损优先级

```
同一根 K 线内：
SL > TP > ENTRY

示例：
价格同时触及 SL 和 TP → SL 优先成交
```
