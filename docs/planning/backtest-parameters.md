## 4. 参数优先级分层

> 目的：避免把“策略真假鉴别器”和“结果放大器”混在同一轮搜索/验证中。

### 4.1 第一优先级：结构参数（先验证）

这层决定策略结构是否成立，优先级高于普通过滤参数。

| 参数 | 当前状态 | 优先级 | 原因 |
|------|----------|--------|------|
| direction / LONG-only | 已锁定 | P0 | 决定是否允许 SHORT，属于结构开关 |
| breakeven_enabled | 已锁定但待复核 | P0 | 直接改变 TP1 后剩余仓位的生存方式 |
| tp_targets | 已锁定 | P0 | 决定盈亏比结构，已验证对结果影响极大 |
| tp_ratios | 已锁定 | P0 | 决定利润兑现节奏，与 tp_targets 成组出现 |

### 4.2 第二优先级：信号过滤参数（结构稳定后再打磨）

这层决定“什么样的 K 线能进场”，适合在结构层稳定后继续优化。

| 参数 | 当前状态 | 优先级 | 原因 |
|------|----------|--------|------|
| ema_period | 已搜索，最优 50 | P1 | 高敏感，决定趋势过滤快慢 |
| min_distance_pct | 已搜索，最优 0.005 | P1 | 决定价格脱离 EMA 的最小强度 |
| mtf_ema_period | 未搜索，沿用 system config | P1.5 | 重要，但更接近策略定义，不宜过早混入搜索 |
| Pinbar 细参数 | 未搜索，沿用默认 | P2 | 会改信号定义，暂不优先 |
| max_atr_ratio | 已验证冗余 | P3 | 当前证据显示对结果无实质影响 |

### 4.3 第三优先级：资金管理参数（最后再碰）

这层更像“结果放大器”，主要影响仓位和波动，不适合用来先验证 alpha。

| 参数 | 当前状态 | 优先级 | 原因 |
|------|----------|--------|------|
| max_loss_percent | 未搜索，默认 1% | P2 | 影响单笔亏损幅度，不直接证明策略更对 |
| max_leverage | 未搜索，默认 10 | P2 | 影响放大倍数，更偏资金管理 |
| max_total_exposure | 未搜索，默认 80% | P2 | 影响组合敞口，不属于信号/出场逻辑 |
| max_position_percent | 未搜索，默认 20% | P2 | 影响单次下注大小，不属于 alpha 逻辑 |
| daily_max_loss / daily_max_trades / min_balance | 未搜索 | P2 | 属于风控保护网，应与策略搜索分离 |

### 4.4 当前推荐验证顺序

1. `LONG-only`
2. `breakeven_enabled`
3. `tp_targets`
4. `tp_ratios`
5. `ema_period`
6. `min_distance_pct`
7. `mtf_ema_period`
8. Pinbar 细参数
9. 风控 / 杠杆 / 单笔损失

一句话原则：

- 先验证“方向和出场”
- 再验证“信号过滤”
- 最后才碰“仓位和杠杆”

## 4.5 后续参数回测路线图（建议执行顺序）

> 目的：避免再次回到“所有参数一起搜”的黑盒撞运气方式。后续回测应按因果层级推进，而不是按参数数量推进。

### 第一阶段：先守住结构层，不轻易重开

当前结构层默认保持冻结：

- `direction = LONG-only`
- `breakeven_enabled = False`
- `tp_targets = [1.0, 3.5]`
- `tp_ratios = [0.5, 0.5]`

原因：

1. 这层定义的是策略本体赚什么钱，不是普通微调参数。
2. 现有证据已说明：
   - `BE=True` 不能救 `2023`
   - 更防守型 exit 不能解决 market regime 不匹配
3. 若这一层再次频繁搜索，等于把“顺大逆小抓趋势延续”和“别的策略物种”混在一起。

因此：

- 除非明确要验证“策略物种是否改变”，否则这层不作为当前主搜索对象。

### 第二阶段：优先搜索信号过滤层

这是当前最值得继续优化的主战场。

建议优先顺序：

1. `ema_period`
2. `min_distance_pct`
3. `mtf_ema_period`
4. `Pinbar` 细参数

当前建议：

- `ema_period`：先做小范围单变量敏感性
  - 建议档位：`40 / 50 / 60 / 70 / 90`
- `min_distance_pct`：在当前基线附近微调
  - 建议档位：`0.004 / 0.005 / 0.006 / 0.008`
- `mtf_ema_period`：在主周期过滤稳定后再测
  - 建议档位：`40 / 60 / 80`

说明：

1. `ema_period` 当前已证实高敏感，仍是首要搜索对象。
2. `min_distance_pct` 代表“离 EMA 多远才算有意义的回撤结束”，适合与 `ema_period` 联动验证。
3. `mtf_ema_period` 更接近“顺大”的定义，不宜过早混入第一轮大搜索。
4. `Pinbar` 细参数会直接改变信号定义，属于后置验证项。

### 第三阶段：把“市场适配”当成独立研究层

后续不应只问“什么参数更赚钱”，还应问：

- 这条策略在什么市场状态下该开？
- 什么市场状态下不该开？

建议把以下内容视为独立分析主题，而不是普通参数搜索：

- higher timeframe EMA slope
- trend persistence / follow-through 强度
- 趋势连续性
- 波动扩张 / 收敛状态

原因：

1. `2023` 的失败更像 `LONG-only + regime mismatch`，而不是单点参数失配。
2. 因此后续最值钱的验证，不一定是继续调 `EMA`，而是确认：
   - 当前市场是否有足够的延续土壤
   - `Pinbar` 抓到的是“真延续”还是“弱反弹”

这一层的核心目标不是：

- 让所有年份都盈利

而是：

- 识别这条策略的适用边界

### 第四阶段：最后才碰资金表达层

这层包括：

- `max_loss_percent`
- `max_total_exposure`
- `max_leverage`
- `max_position_percent`
- `daily_max_loss`
- `daily_max_trades`

原则：

1. 这层更像收益/回撤放大器，不适合用来先验证 alpha。
2. 若过早把这层和信号层混搜，很容易把“资金表达结果”误判成“策略本体发现”。
3. 当前更合理的顺序是：
   - 先证明信号过滤是否合理
   - 再证明市场适配边界
   - 最后再决定怎么放大

### 推荐的回测方法顺序

#### 方法 1：单变量敏感性

先固定当前基线，只动一个参数。

优先做：

- `ema_period`
- `min_distance_pct`
- `mtf_ema_period`

目的：

- 看参数是否存在稳定区
- 看最优值是否只是偶然尖点

#### 方法 2：双变量网格

只做有因果关系的组合，不做全参数混合爆搜。

优先组合：

- `ema_period × min_distance_pct`
- `ema_period × mtf_ema_period`

不建议一开始就做：

- `EMA × TP × Exposure × Risk` 四维混搜

#### 方法 3：边界验证

后续每轮参数搜索，不应只看总收益，还应固定检查：

- `2023 / 2024 / 2025` 分年表现
- `MFE / MAE`
- `first-touch`
- `+1R / +2R / +3.5R` 可达率
- 交易次数与持仓重叠情况

目标：

- 判断“参数更好”是因为信号质量更高，还是只是碰巧更适配某一年

#### 方法 4：资金层后置验证

结构层和过滤层稳定后，再做：

- `1.0x / 1.5x / 2.0x exposure`
- `1% / 1.5% / 2% max_loss_percent`

并明确把这一轮的目的定义为：

- 资金表达测试

而不是：

- 策略真假验证

### 当前明确不建议优先搜索的项

1. `daily_max_trades`
2. `max_leverage`
3. `ATR`（当前证据已偏向冗余）
4. 细粒度 trailing 参数
5. 同时改 `TP + EMA + Exposure + Risk` 的混合搜索

原因：

- 这些项要么当前证据价值较低，要么很容易把搜索结果变成不可解释的混合产物。

### 一句话路线图

后续参数回测建议固定为：

1. 先锁结构层
2. 再搜信号过滤层
3. 再研究市场适配边界
4. 最后才碰资金表达层

一句话原则：

- 不是“哪个参数都能搜”
- 而是“先搜最接近策略本质的那一层”

## 11. 关键验证结论

### 11.1 结构层验证（已完成）

1. **LONG-only 优于双向**：3年总 PnL 改善 +5348 USDT（+175%）

2. **币种/周期适用性**：仅 ETH 1h 适用
3. **最优参数组合**：ema=50, min_distance_pct=0.005, tp=[1.0,3.5], tp_ratios=[0.5,0.5]

### 11.3 资金层验证（已完成）

2. **选择建议**：
   - 推荐默认档用于生产环境
   - 激进档仅用于短期高波动市场或测试环境

### 11.4 完整基线结构（已冻结）
