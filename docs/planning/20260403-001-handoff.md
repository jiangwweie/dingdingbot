### 决策 1: SQLite 单例约束改用触发器实现

- **背景**: 架构设计文档中使用 `CHECK (id = 1)` 约束，但这在 SQLite 中只是应用层约束，INSERT 时不会强制检查
- **选择方案**: BEFORE INSERT/UPDATE 触发器
- **拒绝方案**: CHECK 约束 + 部分索引
- **理由**: 触发器在 SQLite 中是强制的，INSERT 时会直接 ABORT
- **影响**: 需要为 `risk_configs` 和 `system_configs` 各创建 3 个触发器
- **后续**: 策略激活单例约束也改用触发器（2 个触发器）

### 决策 2: 热重载安全协议五步流程

- **背景**: 原设计未考虑正在处理 K 线时热重载的状态不一致风险
- **选择方案**: 暂停→等待→重建→恢复 五步协议
- **拒绝方案**: 直接热重载（简单但危险）
- **理由**: 避免信号处理中断导致状态不一致
- **影响**: `apply_config()` 方法需要增加暂停/恢复逻辑
- **后续**: 前端需要禁用热重载按钮，等待处理完成
