成功完成 P0 级别关键缺陷修复，解决 WebSocket K线选择逻辑错误问题。

**问题背景**：
- WebSocket 使用 `ohlcv[-1]`（最新未收盘 K线）进行形态检测
- 导致信号在 K线未收盘时就触发，造成误判
- 极小波幅 K线被误判为 Pinbar 形态

**修复方案**：
- ✅ 优先使用交易所 `x` 字段判断 K线收盘状态
- ✅ 后备使用时间戳推断机制
- ✅ 增加 Pinbar 最小波幅检查（ATR 10% 动态阈值）
- ✅ 完整的架构设计文档（ADR + 契约表 + 实施清单）

## 📝 关键技术决策

### 决策 1：WebSocket K线选择策略

**选择**: 交易所 `x` 字段优先 + 时间戳后备

**理由**:
- 交易所官方契约，准确性最高
- 兼容不支持 `x` 字段的交易所
- 多层防御机制

**实现**:
```python
# 优先检查 x 字段
if raw_info and 'x' in raw_info:
    is_closed = bool(raw_info['x'])
    if is_closed:
        # x=true → ohlcv[-1] 是刚收盘的 K线
        return ohlcv[-1]
else:
    # 后备：时间戳推断
    # 使用 ohlcv[-2]（前一根已收盘 K线）
    return ohlcv[-2]
```

---

### 决策 2：Pinbar 最小波幅检查

**选择**: 动态 ATR 10% + 固定 0.5 USDT 后备

**理由**:
- 自适应不同币种和时间段
- 防止开盘初期极小波幅误判
- ATR 不可用时使用固定阈值

**实现**:
```python
if atr_value and atr_value > 0:
    min_range = atr_value * Decimal("0.1")  # ATR 的 10%
else:
    min_range = Decimal("0.5")  # 固定后备值

if candle_range < min_range:
    return None  # 跳过极小波幅
```
