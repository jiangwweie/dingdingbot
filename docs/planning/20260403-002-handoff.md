## 💡 技术决策记录

### 决策 1: SQLite 单例约束改用触发器实现
- **背景**: 架构设计文档中使用 `CHECK (id = 1)` 约束，但这在 SQLite 中只是应用层约束
- **选择方案**: BEFORE INSERT/UPDATE 触发器
- **理由**: 触发器在 SQLite 中是强制的，INSERT 时会直接 ABORT
- **影响**: `risk_configs` 和 `system_configs` 各创建 3 个触发器

### 决策 2: 热重载安全协议五步流程
- **背景**: 原设计未考虑正在处理 K 线时热重载的状态不一致风险
- **选择方案**: 暂停→等待→重建→恢复 五步协议
- **理由**: 避免信号处理中断导致状态不一致

### 决策 3: ConfigManager 异步初始化
- **背景**: 从 YAML 加载改为 DB 加载，需要异步 IO
- **选择方案**: `async initialize()` + `async load_all_from_db()`
- **影响**: 测试文件需适配新接口
