# Blocker Classification Contract

## Purpose

This contract defines how main control names blockers while moving a selected
StrategyGroup from admission toward the official live-submit path.

The project does not need more broad explanations for why no trade happened.
The current operating target is Pre-Trade Runtime V0:

```text
active StrategyGroup candidate symbol set
-> classify per-symbol readiness blockers precisely
-> remove engineering / scope / policy / runtime blockers
-> promote fresh satisfied candidates without authority leakage
-> narrow at most one candidate into action-time lane input
-> refresh action-time facts
-> candidate / authorization evidence
-> FinalGate
-> Operation Layer
-> protection / reconciliation / settlement / review
```

Generated artifacts, replay summaries, daily status reports, read-only watcher
expansions, and review ledgers are valid only when they move a lane to a later
Live Enablement state or prove a blocker with per-symbol / per-fact evidence.
They are not independent completion paths.

## Core Rule

Every blocker must name the earliest missing state that prevents an active
StrategyGroup candidate symbol from advancing toward pre-trade readiness,
promotion, or the narrowed action-time lane.

Do not use broad labels such as `waiting_for_market`,
`fresh_signal_absent`, `missing_fact`, or `live_detector_artifact_missing`
unless the stricter class below is actually true.

`market_wait_validated` is allowed only after all non-market blockers for that
candidate symbol have been closed or explicitly scoped out by Owner policy.

## Blocker Classes

| Class | Exact meaning | Owner | Next action |
| --- | --- | --- | --- |
| `artifact_missing` | Required current artifact is absent, unreadable, or not generated by the current monitor sequence | engineering | Generate or wire the artifact through the standard monitor path |
| `schema_invalid` | Artifact exists but required identity, schema version, field type, or row shape is invalid | engineering | Fix producer schema and add regression coverage |
| `detector_not_attached` | Strategy has no live/current detector attached to the selected lane | engineering | Attach detector to monitor sequence and parity audit |
| `watcher_tick_missing` | Detector exists but no current watcher tick or candle/public-fact input is available for the lane | runtime | Refresh or repair watcher/public facts source |
| `scope_not_attached` | Strategy/symbol/side/timeframe exists only in research or read-only review and is not attached to selected runtime observation scope | engineering / owner | Produce scoped live-observation or live-scope proposal with policy boundary |
| `computed_not_satisfied` | Detector and facts ran successfully, but one or more strategy facts are false, stale, or below threshold | market | Continue observation; report failing fact matrix |
| `replay_live_rule_mismatch` | Replay event cannot be reproduced because replay rules and current live detector/fact rules differ | engineering / strategy_review | Normalize rules or record a strategy revision decision |
| `action_time_boundary_not_reproduced` | A live-like detector event cannot reach candidate/auth or action-time rehearsal without exchange write | engineering / runtime | Repair non-executing action-time rehearsal path |
| `policy_scope_missing` | Owner capital/profile/symbol/side/leverage/loss-unit/attempt-cap/tier policy is missing for the lane | owner | Ask for the specific scoped policy decision |
| `runtime_profile_scope_missing` | Runtime profile, selected StrategyGroup binding, symbol lane, side, leverage, notional, or exposure scope is absent or inconsistent | runtime / engineering | Bind or repair runtime profile scope without expanding authority silently |
| `market_wait_validated` | All non-market blockers are closed and the only missing condition is a current fresh eligible signal | market | Keep watching; preempt lower work on fresh signal |
| `action_time_preflight_ready` | A fresh satisfied lane has active server runtime coverage, action-time path readiness, public facts, private action-time facts, account/balance facts, and active-position/open-order clearance for non-executing preflight input generation | engineering | Prepare non-executing FinalGate preflight input; this does not grant FinalGate, Operation Layer, or exchange-write authority |
| `active_position_resolution` | Active position, open order, or protection state conflicts with new submit | runtime / safety | Reconcile or close via official recovery path |
| `hard_safety_stop` | Execution would violate FinalGate, Operation Layer, stale fact, missing protection, duplicate-submit, wrong-account, withdrawal, transfer, credential, profile, sizing, or destructive boundary | safety | Stop and surface safety state |
| `review_only_warning` | Evidence quality concern that may affect promote/revise/park/kill decisions but does not block non-executing engineering closure | strategy_review | Record strategy review impact without blocking live path engineering |

Legacy labels may appear in generated views only when mapped to one of these
classes before planning, task acceptance, or Owner-facing state interpretation.

## Valid Market Wait

`market_wait_validated` requires all of the following:

| Requirement | Required state |
| --- | --- |
| Asset admission | StrategyGroup is a final-owned admitted trial/runtime asset or has an explicitly selected trial-admission lane |
| Scope | StrategyGroup, symbol, side, timeframe, and runtime observation scope are attached or explicitly deferred |
| Policy | Owner policy covers capital, profile, loss unit, attempt cap, and symbol/side boundary when needed |
| Detector | Current live/current detector is attached to the lane |
| Watcher input | Current watcher tick or public facts source is present |
| Facts | Required facts are computed and classified per symbol / per fact |
| Classification | The blocker matrix distinguishes missing artifact, missing scope, no tick, false computed fact, and rule mismatch |
| Action-time path | If a fresh signal appears, candidate/auth and action-time rehearsal path can continue without redefining architecture |
| Fresh signal | No current fresh eligible signal exists |

If any row is false, the blocker is not market wait. Use the earliest concrete
class from the table above.

## Replay And Live Boundary

Replay is historical. Live cannot capture historical opportunities.

Replay exists to calibrate rules, detector parity, fact mapping, and action-time
rehearsal so the live system can catch current and future opportunities. A
replay signal may prove that a live detector should have a rule, scope, or fact
mapping. It must never become:

- live RequiredFacts;
- FinalGate input;
- Operation Layer evidence;
- exchange-write authority;
- proof that a current market opportunity exists.

## Per-Symbol / Per-Fact Matrix

Detector, watcher, replay-live parity, and daily status outputs must preserve a
matrix with at least:

| Field | Meaning |
| --- | --- |
| `strategy_group_id` | StrategyGroup under review |
| `symbol` | Runtime or proposed symbol lane |
| `side` | Long/short or strategy-specific side |
| `timeframe` | Detector / replay timeframe |
| `detector_attached` | Whether current detector exists |
| `watcher_tick_present` | Whether current input exists |
| `computed` | Whether facts were computed |
| `fresh_signal_present` | Whether current fresh signal exists |
| `failed_facts` | Exact facts that were false, stale, or missing |
| `blocker_class` | One class from this contract |
| `next_action` | One action that removes or validates the blocker |

Do not report `live_detector_artifact_missing` or equivalent when an artifact
exists, the detector ran, watcher input is present, and the computed facts are
simply not satisfied.

## Read-Only Scope Rule

Read-only watcher expansion is an observation tool, not a milestone.

For the current live-enablement phase, read-only scope is accepted only when it
does one of these:

- produces a live-scope proposal for specific StrategyGroup + symbol lanes;
- proves `scope_not_attached` with per-symbol evidence;
- proves `computed_not_satisfied` for current/future opportunity waiting;
- proves `replay_live_rule_mismatch` and creates a bounded repair task;
- is explicitly kept as later strategy-observation work outside the current
  live-submit advancement lane.

It must not be counted as completion merely because more symbols are watched.

## Completion Rule

A task is complete only when it does one of these:

- removes a blocker and moves a lane to a later Live Enablement state;
- reclassifies a broad blocker into a precise class with per-symbol / per-fact
  evidence and a machine-checkable next action;
- proves `market_wait_validated` using the full valid-market-wait checklist;
- stops at an explicit Owner policy or hard safety boundary that cannot be
  removed by engineering.

A task is not complete when it only creates a report, projection, readiness
artifact, audit explanation, or no-trade narrative without changing the blocker
state.

## Authority Boundary

This contract does not authorize:

- FinalGate bypass;
- Operation Layer bypass;
- live profile expansion;
- sizing-default expansion;
- exchange write from replay, synthetic, read-only, or audit evidence;
- stale-fact execution;
- missing-protection execution;
- duplicate submit;
- conflicting position/open-order submit;
- withdrawal or transfer;
- credential mutation;
- destructive production cleanup.

It only defines the blocker language that must drive planning, implementation,
review, and Owner-facing summaries.
