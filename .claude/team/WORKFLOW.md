## 项目级规范 (所有角色遵守)

### 🟢 开工前检查清单 (Pre-Flight)

**所有角色在开始任务前必须完成**:

```markdown
- [ ] **理解需求**: 已阅读并复述任务目标
- [ ] **确认边界**: 明确任务范围和交付物
- [ ] **检查依赖**: 确认前置任务已完成或无需依赖
- [ ] **环境准备**: 本地开发环境已就绪
```

**开工流程**:

```
1. 阅读任务描述 → 2. 复述理解 → 3. 确认无疑问 → 4. 开始执行
```

---

### 🔴 收工检查清单 (Post-Flight)

**所有角色在完成任务后必须完成**:

```markdown
- [ ] **自验证通过**: 本地测试/运行正常
- [ ] **代码格式化**: 遵循项目代码规范
- [ ] **无 TODO 遗留**: 或已明确标注待办事项
- [ ] **提交规范**: Git commit message 符合约定式规范
- [ ] **文档更新**: 相关文档已同步更新（如适用）
```

**提交前验证命令**:

```bash
# 后端验证
pytest tests/unit/ -v --tb=short
black src/ tests/
mypy src/

# 前端验证
cd web-front && npm run build
npm run lint
```

## 开工/收工流程示例

### 场景 1: 简单任务 (单文件修改)

```
【开工】
1. 阅读任务 → 2. 确认范围 → 3. 开始修改

【收工】
1. 本地验证 → 2. 提交代码 → 3. 推送远程
```

### 场景 2: 中等任务 (2-3 文件)

```
【开工】
1. 阅读任务 → 2. 创建简易计划 → 3. 确认依赖 → 4. 开始执行

【收工】
1. 运行测试 → 2. 更新进度 → 3. 提交代码 → 4. 推送远程
```

### 场景 3: 复杂任务 (前端 + 后端 + 测试)

```
【开工】(Coordinator)
1. 调用 planning-with-files-zh
2. 创建 task_plan.md
3. 分解任务 (TaskCreate)
4. 分配角色 (Agent 并行调度)

【执行】
各角色按专属规范执行

【收工】(Coordinator)
1. 集成验证 (所有测试通过)
2. 生成交付报告
3. 更新 progress.md
4. 提交并推送
```


## 违规处理

| 违规类型 | 处理方式 |
|----------|----------|
| 未开工检查 | 提醒补充，任务标记为"需改进" |
| 未收工验证 | 测试失败则返回修复 |
| 文档未更新 | 创建待办任务补充 |
